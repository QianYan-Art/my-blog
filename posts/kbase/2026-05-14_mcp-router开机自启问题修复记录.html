MCP-Router开机自启问题修复记录

2026-05-144 min

一、问题现象

  • 本机安装 MCP Router 后，每次启动程序都会自动把自己加入 Windows 开机启动项。
  • 即使在任务管理器中手动禁用启动项，只要再次打开 MCP Router，启动项又会被重新写回。
  • 之前曾临时放置过一个“删除自启项”的脚本，但该脚本只能在登录时补删一次，无法阻止程序再次写回自启。

二、根因结论

排查 MCP Router 本地安装包后，确认问题不在 Windows 本身，而在 MCP Router 程序逻辑：

  • 程序主进程中存在 setLoginItemSettings / openAtLogin 相关逻辑。
  • showWindowOnStartup 配置项实际只影响“开机时是否显示主窗口”。
  • 程序在启动过程中会把 openAtLogin 固定设为 true，导致手动禁用启动项后仍会被再次写回。

结论：

  • 这是 MCP Router 的应用逻辑问题，不是普通启动项残留问题。
  • 单纯删除注册表启动项，只能治标，不能治本。

三、处理过程

1. 已验证的无效方案

  • 启动文件夹脚本：

    • - 登录时删除 HKCU\Software\Microsoft\Windows\CurrentVersion\Run 中的 com.squirrel.MCP-Router.MCPRouter - 该方案只能在登录时执行一次 - 用户后续手动打开 MCP Router 后，启动项仍会被重新写回

2. 中途尝试但已回滚的方案

  • 曾尝试直接修改 MCP Router 安装包内的 app.asar
  • 该方案导致程序无法正常打开
  • 已将安装包恢复到原始可运行状态
  • 后续未继续采用安装包篡改方式

3. 最终采用方案

采用“外部监视压制”方案，不再修改 MCP Router 安装包：

  • 创建后台监视脚本，持续监视 MCP Router 的开机自启注册表项
  • 一旦检测到 com.squirrel.MCP-Router.MCPRouter 被重新写回：

    • - 立即删除 HKCU\Software\Microsoft\Windows\CurrentVersion\Run 中的对应值 - 同时删除 HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\StartupApproved\Run 中的对应值

  • 对于带 --hidden 或 --minimized 参数启动的隐藏自启进程，脚本会自动结束该进程
  • 用户手动正常打开的 MCP Router 主进程不会被误杀

四、当前保留文件

1. 后台监视脚本

路径：

C:\Users\lenovo\AppData\Local\MCP-Router\watch-mcp-router-autostart.ps1

作用：

  • 每 2 秒轮询一次
  • 自动删除 MCP Router 写回的自启注册表项
  • 自动结束隐藏启动的 MCP Router.exe

2. 登录启动器

路径：

C:\Users\lenovo\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\watch-mcp-router-autostart.vbs

作用：

  • 在用户登录 Windows 后，静默拉起上述 PowerShell 监视脚本

五、当前状态

截至 2026-05-14，已确认以下状态：

  • MCP Router 可正常打开
  • mcp-router 工具链可正常工作
  • Tavily 已在当前 Codex 会话中成功调用
  • MCP Router 启动项会被后台监视器自动清除
  • 用户手动打开 MCP Router 后，任务管理器中的启动项可在短时间内自动消失

六、已删除内容

以下临时排障残留已删除：

  • 早期登录补刀脚本 disable-mcp-router-autostart.cmd
  • 本地补刀脚本 disable-autostart.cmd
  • 修改安装包时留下的备份文件 app.asar.bak
  • 修改安装包时留下的备份文件 index.js.bak

七、后续注意事项

1. 不要手动删除以下两个文件

  • C:\Users\lenovo\AppData\Local\MCP-Router\watch-mcp-router-autostart.ps1
  • C:\Users\lenovo\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\watch-mcp-router-autostart.vbs

如果删除，MCP Router 的开机自启问题大概率会再次出现。

2. 后续如 MCP Router 升级

当前方案属于外部压制，不依赖旧版安装包，因此软件升级后大概率仍然有效。

但如果升级后：

  • 启动参数改变
  • 进程名改变
  • 自启注册表键名改变

则需要重新检查监视脚本中的判断逻辑。

3. 如果后续准备彻底卸载该方案

需要同时删除：

  • C:\Users\lenovo\AppData\Local\MCP-Router\watch-mcp-router-autostart.ps1
  • C:\Users\lenovo\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\watch-mcp-router-autostart.vbs

然后重新观察 MCP Router 是否再次写回开机启动项。

八、建议

  • 当前无需继续清理
  • 维持现状即可
  • 若后续 MCP Router 官方修复该问题，再考虑移除本地监视脚本
