安装 Rclone

2025-11-206 min

适用环境：Ubuntu 22.04 / 10GB 内存 / 无桌面环境 (Headless)

目标用户：qianyan

挂载路径：/home/gdrive

第一步：安装 Rclone 与依赖

在服务器上执行以下命令安装 Rclone 官方版本及 FUSE 3 文件系统支持：

# 安装 Rclone
sudo -v ; curl [https://rclone.org/install.sh](https://rclone.org/install.sh) | sudo bash

# 安装 FUSE3 (Ubuntu 22.04 必须)
sudo apt update
sudo apt install fuse3 -y

第二步：配置连接 (Remote)

由于服务器没有浏览器，我们需要配合本地电脑完成授权。

  1. 在服务器终端输入：
    rclone config

- 输入 n 新建。

- name: 输入 gdrive (或者你喜欢的名字)。

- Storage: 选择 Google Drive (通常是 drive)。

- client_id / client_secret: 留空回车。

- Scope: 选 1 (Full access)。

- service_account_file: 留空回车。

- Edit advanced config: n。

- Use auto config?: 选 n (No) (关键步骤)。

  1. 在本地电脑上：

- 下载并解压 Rclone (Windows/Mac)。

- 在本地终端运行服务器给出的命令：rclone authorize "drive" "xxxxx..."。

- 浏览器登录授权后，复制终端里生成的 Token 代码。

  1. 回到服务器终端：

- 粘贴 Token 代码。

- Team Drive 选 n (除非你是团队盘)。

- 输入 y 确认配置，q 退出。

第三步：准备挂载目录与权限

为了避免 fusermount: user has no write access 错误，必须正确设置目录权限。

# 1. 创建挂载点
sudo mkdir -p /home/gdrive

# 2. 修改目录所有权为当前用户 (qianyan)
sudo chown qianyan:qianyan /home/gdrive

# 3. 开启 FUSE 的 allow_other 权限
sudo sed -i 's/#user_allow_other/user_allow_other/g' /etc/fuse.conf

第四步：配置 Systemd 开机自启服务

创建后台服务，确保挂载稳定且开机自启。

1. 创建服务文件

sudo nano /etc/systemd/system/rclone-gdrive.service
  1. 填入配置内容

(针对 10GB 内存优化，高性能模式)

[Unit]
Description=Rclone Google Drive Mount
Wants=network-online.target
After=network-online.target

[Service]
Type=notify
User=qianyan
Group=qianyan
# 环境变量指向你的配置文件路径
Environment=RCLONE_CONFIG=/home/qianyan/.config/rclone/rclone.conf

KillMode=none
Restart=on-failure
RestartSec=5

# --- 核心挂载命令 ---
ExecStart=/usr/bin/rclone mount gdrive: /home/gdrive \
--allow-other \
--allow-non-empty \
--vfs-cache-mode full \
--vfs-cache-max-size 10G \
--vfs-read-chunk-size 32M \
--vfs-read-chunk-size-limit 1G \
--buffer-size 256M \
--umask 002 \
--log-level INFO

# --- 卸载命令 ---
ExecStop=/bin/fusermount3 -u /home/gdrive

[Install]
WantedBy=multi-user.target

_> 注意：如果后台有 Emby 等大量扫描任务导致内存吃紧，请将 --buffer-size 256M 改为 64M。_

3. 启动服务

# 重载配置
sudo systemctl daemon-reload
# 启动
sudo systemctl start rclone-gdrive
# 开机自启
sudo systemctl enable rclone-gdrive
# 检查状态 (应显示 active running)
systemctl status rclone-gdrive

第五步：Docker 挂载使用技巧

如果你需要在 Docker 容器（如 Emby, FileBrowser, QBittorrent）中使用这个挂载点，必须使用 :rslave 模式，否则宿主机重挂载后容器会断连。

Docker Compose 示例：

volumes:
  - /home/gdrive:/data:rslave

Docker Run 示例：

docker run -v /home/gdrive:/data:rslave ...

附录：如何彻底删除/卸载

如果未来你想移除这个挂载并清理系统，请按顺序执行：

  1. 停止并禁用服务：
    sudo systemctl stop rclone-gdrive
    sudo systemctl disable rclone-gdrive
  1. 删除服务文件：
    sudo rm /etc/systemd/system/rclone-gdrive.service
    sudo systemctl daemon-reload
  1. 确保已卸载：
    fusermount3 -u /home/gdrive
    # 如果提示未找到挂载点，说明已经卸载干净了
  1. 删除配置文件和缓存：
    # 删除配置 (Token 等信息)
    rm -rf /home/qianyan/.config/rclone
    # 删除缓存数据
    rm -rf /home/qianyan/.cache/rclone
  1. 删除挂载目录 (可选)：
    # 只有确定里面全是空的或者是云端文件（已卸载状态）才删
    sudo rmdir /home/gdrive
  1. 卸载 Rclone 程序 (可选)：
    sudo rm /usr/bin/rclone
    sudo rm /usr/local/share/man/man1/rclone.1
