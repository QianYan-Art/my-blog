本文档详细介绍如何通过 frp 将本地内网服务暴露到公网，包含部署、管理和删除全流程。

一、环境说明

  • 服务端：阿里云 Ubuntu 服务器（公网 IP: 101.200.155.141）
  • 客户端：Windows 本地电脑
  • 目标：将本地 1717 端口的 EasyDataset 服务通过 8080 端口暴露到公网，并启用密码保护

二、服务端部署

1. 下载并解压 frp

# 进入用户目录
cd ~

# 创建项目目录
mkdir -p Frp && cd Frp

# 下载 frp 0.65.0 Linux 版本
wget https://github.com/fatedier/frp/releases/download/v0.65.0/frp_0.65.0_linux_amd64.tar.gz

# 解压
tar -xzf frp_0.65.0_linux_amd64.tar.gz
cd frp_0.65.0_linux_amd64

2. 配置 frps.toml

创建服务端配置文件：

nano frps.toml

写入以下内容：

bindPort = 7000          # frp 通信端口
vhostHTTPPort = 8080     # HTTP 代理端口，用于访问内网服务

3. 移动文件到系统目录

# 创建系统目录
sudo mkdir -p /etc/frp /usr/local/bin/frp

# 移动配置文件
sudo cp frps.toml /etc/frp/

# 移动可执行文件并添加权限
sudo cp frps /usr/local/bin/frp/
sudo chmod +x /usr/local/bin/frp/frps

4. 创建 systemd 服务

创建服务文件：

sudo nano /etc/systemd/system/frps.service

写入以下内容：

[Unit]
Description=FRP Server Service
After=network.target

[Service]
Type=simple
Restart=always
RestartSec=5s
ExecStart=/usr/local/bin/frp/frps -c /etc/frp/frps.toml

# 可选日志配置，需提前创建文件并授权
# StandardOutput=append:/var/log/frps.log
# StandardError=append:/var/log/frps.log

[Install]
WantedBy=multi-user.target

5. 启动并设置开机自启

# 重载 systemd
sudo systemctl daemon-reload

# 设置开机自启
sudo systemctl enable frps

# 启动服务
sudo systemctl start frps

# 查看状态（确认 active running）
sudo systemctl status frps

6. 验证服务

# 检查进程
ps aux | grep frps

# 检查端口监听（7000 和 8080）
netstat -tlnp | grep -E '7000|8080'

三、客户端配置

1. 下载 Windows 版本

在本地 Windows 电脑访问 frp releases，下载 frp_0.65.0_windows_amd64.zip，解压到任意目录（如 C:\frp）。

2. 配置 frpc.toml

在解压目录创建 frpc.toml：

serverAddr = "101.200.155.141"
serverPort = 7000
auth.token = "123456"  # 如果服务端设置了 token

[[proxies]]
name = "easy-dataset"
type = "http"
localIP = "127.0.0.1"
localPort = 1717
customDomains = ["101.200.155.141"]  # 服务端公网IP或域名
httpUser = "easy-dataset"
httpPassword = "Woaizuoshujuji"

3. 启动 frpc

# PowerShell 或 CMD
frpc -c frpc.toml

看到 start proxy success 即表示成功。

四、防火墙配置

1. 阿里云安全组

登录阿里云控制台，为 ECS 实例添加安全组规则：

方向协议类型端口范围授权对象
入方向自定义 TCP7000/70000.0.0.0/0
入方向自定义 TCP8080/80800.0.0.0/0

2. 系统防火墙（如有）

sudo ufw allow 7000/tcp
sudo ufw allow 8080/tcp

五、域名配置（可选）

Cloudflare 配置

  1. 添加 DNS 记录

    2. - 类型：A - 名称：dataset（子域名） - IPv4 地址：101.200.155.141 - 代理状态：DNS only（点击橙云图标变灰云） - TTL：Auto

  1. 修改 frpc.toml
   customDomains = ["dataset.yourdomain.com"]
  1. 重启 frpc
   frpc -c frpc.toml
  1. 访问地址
   http://dataset.yourdomain.com:8080

⚠️ 重要：中国大陆服务器需备案域名，否则会被拦截。如未备案，请直接使用 IP 访问。

七、管理命令

服务端管理

# 查看实时日志
sudo journalctl -u frps -f

# 查看历史日志
sudo journalctl -u frps --since "1 hour ago"

# 重启服务（修改配置后）
sudo systemctl restart frps

# 停止服务
sudo systemctl stop frps

# 禁用开机自启
sudo systemctl disable frps

客户端管理

Windows 下直接关闭 frpc.exe 窗口即可停止。

八、删除 frp 服务

1. 停止并禁用服务

sudo systemctl stop frps
sudo systemctl disable frps

2. 删除服务文件

sudo rm /etc/systemd/system/frps.service
sudo systemctl daemon-reload

3. 删除程序文件

# 删除可执行文件和配置
sudo rm -rf /usr/local/bin/frp /etc/frp

# 删除用户目录（可选）
rm -rf ~/Frp

4. 清理防火墙规则

# 删除安全组规则（阿里云控制台）
# 或删除 ufw 规则
sudo ufw delete allow 7000/tcp
sudo ufw delete allow 8080/tcp

5. 清理日志（如有）

sudo rm /var/log/frps.log

九、常见问题

Q1: Permission denied 错误

chmod +x /usr/local/bin/frp/frps

Q2: systemd 启动失败 status=217/USER

编辑 /etc/systemd/system/frps.service，注释掉 User= 行或创建专用用户。

Q3: 浏览器提示 ERR_UNSAFE_PORT

更换端口为非受限端口（8080、8888、18080 等）。

Q4: unmarshal ProxyConfig error

检查字段名是否正确：

  • type = "http" 不支持 remotePort，需在服务端配置 vhostHTTPPort
  • 认证字段必须用驼峰命名 httpUser、httpPassword

Q5: 多人同时编辑冲突

frp 只负责转发，不支持并发控制。多人同时操作会导致数据损坏，建议：

  • 限制单用户使用
  • 或改用支持协作的云端工具
