Linux 服务器网络配置与镜像源管理指南

2025-11-207 min
a-初始化

本文档旨在指导新购 Linux 服务器进行基础的网络配置（DNS/网关）以及软件镜像源的切换与恢复。

第一部分：配置 DNS 和网关

1. 临时修改 DNS (立即生效，重启可能失效)

适用于所有 Linux 发行版。

编辑 /etc/resolv.conf 文件：

vi /etc/resolv.conf

添加或修改 nameserver 行：

nameserver 8.8.8.8
nameserver 114.114.114.114

2. 永久修改 DNS 与网关

A. Ubuntu 18.04+ / Debian 10+ (使用 Netplan)

现代 Debian 系发行版通常使用 Netplan 管理网络。

  1. 查看配置文件名称：

    ls /etc/netplan/
# 通常是 00-installer-config.yaml 或 50-cloud-init.yaml

  2. 编辑配置文件：

    sudo vi /etc/netplan/00-installer-config.yaml

  3. 修改如下 (注意 YAML 缩进必须严格对齐)：

    network:
  ethernets:
    eth0:  # 网卡名称，通过 ip addr 查看
      dhcp4: no
      addresses: [192.168.1.100/24]  # 服务器 IP
      routes:
         - to: default
           via: 192.168.1.1  # 这里是网关地址
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114] # 这里是 DNS
  version: 2

  4. 应用配置：

    sudo netplan apply
B. CentOS 7/8 / Rocky / AlmaLinux (使用 Network Scripts)

  1. 找到网卡配置文件：

    cd /etc/sysconfig/network-scripts/
ls
# 找到对应网卡文件，如 ifcfg-eth0

  2. 编辑文件：

    vi ifcfg-eth0

  3. 确保包含以下内容：

    BOOTPROTO=static       # 静态 IP
ONBOOT=yes             # 开机自启
IPADDR=192.168.1.100   # IP 地址
NETMASK=255.255.255.0  # 子网掩码
GATEWAY=192.168.1.1    #网关地址 (修改这里)
DNS1=8.8.8.8           #主 DNS (修改这里)
DNS2=114.114.114.114   #备用 DNS

  4. 重启网络服务：

    # CentOS 7
systemctl restart network
# CentOS 8 / Rocky
nmcli c reload

第二部分：软件镜像源配置

在操作前，请务必备份当前的源文件。

1. 检查当前源

  • Ubuntu/Debian: cat /etc/apt/sources.list

  • CentOS/RHEL: ls /etc/yum.repos.d/

2. 备份源文件

# Ubuntu/Debian
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak

# CentOS/RHEL
sudo mkdir /etc/yum.repos.d/backup
sudo mv /etc/yum.repos.d/*.repo /etc/yum.repos.d/backup/

3. 场景 A：恢复官方源 (如果当前是第三方修改版)

Ubuntu 官方源内容

编辑 /etc/apt/sources.list，根据你的版本代号（如 20.04 是 focal，22.04 是 jammy），替换以下内容：

deb [http://archive.ubuntu.com/ubuntu/](http://archive.ubuntu.com/ubuntu/) focal main restricted universe multiverse
deb [http://archive.ubuntu.com/ubuntu/](http://archive.ubuntu.com/ubuntu/) focal-security main restricted universe multiverse
deb [http://archive.ubuntu.com/ubuntu/](http://archive.ubuntu.com/ubuntu/) focal-updates main restricted universe multiverse
deb [http://archive.ubuntu.com/ubuntu/](http://archive.ubuntu.com/ubuntu/) focal-backports main restricted universe multiverse
CentOS 官方源

CentOS 7 已停止维护（EOL），官方源已移动到 vault。建议 Centos 用户直接使用国内镜像源或者迁移到 Rocky Linux。

如果是 CentOS Stream 8/9，通常只需重置 yum 配置即可恢复默认（前提是系统内还有默认的 release 包）。

4. 场景 B：更换为国内常用镜像源 (推荐)

国内镜像源速度更快，常用的有 阿里云 (Aliyun)、腾讯云 (Tencent)、清华大学 (TUNA)。

系统一：Ubuntu / Debian

方法：使用 sed 命令一键替换 (以 Ubuntu 替换为阿里云为例)

# 替换主源
sudo sed -i 's/[archive.ubuntu.com/mirrors.aliyun.com/g](https://archive.ubuntu.com/mirrors.aliyun.com/g)' /etc/apt/sources.list
sudo sed -i 's/[security.ubuntu.com/mirrors.aliyun.com/g](https://security.ubuntu.com/mirrors.aliyun.com/g)' /etc/apt/sources.list
sudo sed -i 's/[ports.ubuntu.com/mirrors.aliyun.com/g](https://ports.ubuntu.com/mirrors.aliyun.com/g)' /etc/apt/sources.list

# 更新缓存
sudo apt update

若要换成清华源，将上面的 mirrors.aliyun.com 改为 mirrors.tuna.tsinghua.edu.cn。

若要换成腾讯源，改为 mirrors.cloud.tencent.com。

系统二：CentOS 7 / Rocky Linux / AlmaLinux

方法：下载新的 .repo 文件

这里以 CentOS 7 为例 (注意：CentOS 7 基础源已停止，需使用特定配置，以下为阿里云配置)：

# 1. 下载阿里云的 repo 文件
curl -o /etc/yum.repos.d/CentOS-Base.repo [https://mirrors.aliyun.com/repo/Centos-7.repo](https://mirrors.aliyun.com/repo/Centos-7.repo)

# 2. 如果需要 EPEL 源 (推荐)
curl -o /etc/yum.repos.d/epel.repo [https://mirrors.aliyun.com/repo/epel-7.repo](https://mirrors.aliyun.com/repo/epel-7.repo)

# 3. 清理并生成缓存
yum clean all
yum makecache

如果是 Rocky Linux 8/9:

# 替换为阿里云源
sed -e 's|^mirrorlist=|#mirrorlist=|g' \
    -e 's|^#baseurl=[http://dl.rockylinux.org/$contentdir](http://dl.rockylinux.org/$contentdir)|baseurl=[https://mirrors.aliyun.com/rockylinux](https://mirrors.aliyun.com/rockylinux)|g' \
    -i.bak \
    /etc/yum.repos.d/rocky*.repo

# 更新缓存
dnf makecache
