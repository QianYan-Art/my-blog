5.2 邮件 VPS 读取能力

当前新增了一条“不经代理、直接从 Hermes 读写邮件 VPS”的链路：

  • Hermes 侧包装脚本：/home/hermes/.hermes/bin/mail_vps_fetch.py
  • Hermes 顶层 skill：/home/hermes/.hermes/skills/mail-vps-ops/SKILL.md
  • Hermes 侧受限 SSH key：/home/hermes/.ssh/mail_vps_readonly_ed25519
  • Hermes 侧读写 SSH key：/home/hermes/.ssh/mail_vps_readwrite_ed25519
  • Hermes 侧远程账号：hermesmail@23.238.70.240:23333
  • Hermes 侧读写远程账号：hermesmailrw@23.238.70.240:23333
  • 邮件 VPS：23.238.70.240:23333
  • 邮件 VPS 侧受限入口：

    • - /usr/local/bin/hermes-mail-api.py - /usr/local/bin/hermes-mail-ssh-gateway - /etc/sudoers.d/hermesmail-mail-api

  • 邮件 VPS 侧通用读信脚本：/root/mail_ops/mail_read.py
  • 详细实现已拆到专题文档：

    • - /home/hermes/HERMES_MAIL_VPS_INTEGRATION.md

工作方式：

  1. QQBot 走 terminal 工具调用 mail_vps_fetch.py。
  2. mail_vps_fetch.py 按命令类型选择只读或读写 key，SSH 到 hermesmail 或 hermesmailrw。
  3. 邮件 VPS 上的 authorized_keys 强制执行 /usr/local/bin/hermes-mail-ssh-gateway。
  4. 网关按账号类型只允许白名单命令，再通过 sudo -n /usr/local/bin/hermes-mail-api.py 读取或操作 Maildir。
  5. 读附件时，包装脚本会把附件落到 Hermes 本地缓存，并返回 MEDIA: 标签供 QQ 发送。
  6. 返回结构化 JSON。

为什么当前优先用这条路：

  • Hermes 服务器已经实测可直连邮件 VPS 的 23333/993/587。
  • 不需要在 Hermes 服务器上搭代理。
  • 不需要把每个邮箱密码都存到 Hermes。
  • 未来新增邮箱域名后缀时，只要用户给完整邮箱地址，就能按 Maildir=/var/mail/vmail/%d/%n 继续读，不必修改 Hermes 侧逻辑。
  • 受限 key 已实测不能执行 uname -a 这类无关命令；只读和读写权限已经拆开。

常用调用：

/home/hermes/.hermes/bin/mail_vps_fetch.py list-mailboxes

    • - QQ 无法自动重连、长时间掉线、或日志出现新的 ERROR / CRITICAL / Traceback - 需要再次升级 Hermes、重放定制 patch、或 git/venv/启动流程再次异常 - remote_gptsovits 持续不可用并影响当前实际使用

  • 第 4、5 项继续作为历史残留管理，除非误用或明确需要清理，否则不主动处理。
