RackNerd VPS 初始化配置手册（实战Swap操作）

2025-11-224 min
e-RackNerd

主机概况：

  • 配置：2 vCPU / 2.5 GB RAM / 40 GB SSD

  • 系统：Ubuntu 22.04 LTS

  • 用户：Root

  • 网络别名：RackNerd-DF2 (根据 hostname)

1. 内存与 Swap 策略 (Memory Configuration)

鉴于 2.5G 物理内存对于运行 Docker 或现代应用略显紧张，我们实施了 "保物理、大 Swap" 的策略。

当前状态

  • 物理内存：2.5 GB

  • 虚拟内存 (Swap)：4.0 GB (文件形式)

  • 总可用内存空间：~6.5 GB

已执行的操作

  1. 禁用旧分区：禁用了原本位于 /dev/vda3 的 1.2GB Swap 分区。

  2. 创建新 Swap：在根目录下创建了 /swapfile (4GB)。

  3. 性能优化 (Swappiness)：

    将 vm.swappiness 从默认的 60 调整为 10。

    意义：告诉内核“除非物理内存剩余极少，否则不要使用 Swap”。这能显著减少 SSD 的读写损耗，防止 VPS 因频繁换页而卡顿。

常用检查命令

# 查看内存及 Swap 使用情况
free -h

# 查看当前的 Swappiness 值 (应为 10)
cat /proc/sys/vm/swappiness

2. 存储空间规划 (Storage Layout)

我们将原本闲置的 1.2GB 旧 Swap 分区进行了“废物利用”，将其转换为独立的数据存储盘。

当前挂载结构
设备 挂载点 大小 文件系统 用途
/dev/vda2 / ~38G ext4 操作系统、Docker 镜像、程序代码
/dev/vda3 /mnt/logs 1.2G ext4 专用日志、临时备份、非易失性缓存
/swapfile [Swap] 4.0G swap 虚拟内存

配置文件 (/etc/fstab)

系统已配置开机自动挂载，关键配置如下：

# 根分区
/dev/vda2 / ext4 defaults 0 1
# 独立的日志分区 (原 Swap 分区)
/dev/vda3 /mnt/logs ext4 defaults 0 0
# 新的 Swap 文件
/swapfile none swap sw 0 0

3. 使用指南：如何使用 /mnt/logs

由于 /mnt/logs 只有 1.2G，它非常适合存放文本日志或轻量级备份，但不适合存放 Docker 镜像或大型媒体文件。

场景 A：Node.js / PM2 日志

如果你使用 PM2 部署项目，建议将日志指向此目录，防止日志撑爆主系统盘。

# 启动应用时指定日志路径
pm2 start app.js --log /mnt/logs/app.log

# 或者在 ecosystem.config.js 中配置
module.exports = {
  apps : [{
    name: "my-app",
    script: "./app.js",
    error_file: "/mnt/logs/my-app-err.log",
    out_file: "/mnt/logs/my-app-out.log",
    merge_logs: true
  }]
}

场景 B：Docker 容器日志挂载

运行 Docker 容器时，可以将容器内的日志目录映射出来。

# 示例：运行 Nginx 并将日志存放在独立分区
docker run -d \
  --name nginx-web \
  -v /mnt/logs/nginx:/var/log/nginx \
  nginx

场景 C：定期备份关键配置

可以编写简单的 Crontab 脚本，将重要的配置文件打包到这里。

# 示例：备份 /etc/nginx 配置
tar -czf /mnt/logs/backup_nginx_$(date +%F).tar.gz /etc/nginx

4. 维护与清理建议

由于 1.2G 空间较小，建议定期清理日志。

1. 检查磁盘占用

df -h | grep vda3

2. 快速清空日志文件 (而不删除文件)

如果发现某个日志文件（例如 app.log）太大，可以使用以下命令瞬间清空内容，且不需要停止服务：

truncate -s 0 /mnt/logs/app.log

5. 后续推荐安装 (Next Steps)

既然基础环境已调优，建议安装以下工具以发挥 VPS 最大效能：

  1. Docker & Docker Compose (部署服务的核心)

    curl -fsSL [https://get.docker.com](https://get.docker.com) -o get-docker.sh && sh get-docker.sh

  2. htop (比 top 更好看的监控工具)

    apt install htop -y

    使用 htop 可以直观地看到 4GB Swap 条和 2.5GB 内存条的使用情况。
