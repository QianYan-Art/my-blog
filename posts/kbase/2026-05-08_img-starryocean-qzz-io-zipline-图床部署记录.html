服务器记录

img.starryocean.qzz

2026-05-084 min
f-NetCup

1. 目标

在同一台 netcup Debian 13 服务器上新增一个图床站点：

  • 域名：img.starryocean.qzz.io
  • 用途：个人自用图床
  • 要求：

    • - 本地存储 - HTTPS - 可持续扩展 - 禁止公开注册，只保留现有账号可用

2. 图床选型

最终选择：

  • Zipline

选择原因：

  • Docker 部署简单
  • 支持本地存储
  • 支持文件上传与图片外链
  • 支持仪表盘和用户管理
  • 支持后续接入 ShareX / 脚本上传
  • 文档与社区活跃

参考：

  • GitHub：https://github.com/diced/zipline

3. 目录规划

3.1 应用目录

  • /srv/apps/zipline

3.2 数据目录

  • /srv/data/zipline/uploads
  • /srv/data/zipline/public
  • /srv/data/zipline/themes
  • /srv/data/zipline/postgres

3.3 日志目录

  • Nginx 访问日志：/srv/logs/nginx/img.access.log
  • Nginx 错误日志：/srv/logs/nginx/img.error.log

4. Docker 编排

编排文件：

  • /srv/apps/zipline/compose.yaml

环境变量文件：

  • /srv/apps/zipline/.env

服务组成：

4.1 PostgreSQL

  • 容器名：zipline-postgres
  • 镜像：postgres:16
  • 数据目录：/srv/data/zipline/postgres

4.2 Zipline

  • 容器名：zipline
  • 镜像：ghcr.io/diced/zipline:latest
  • 本地监听：127.0.0.1:3000
  • 上传目录：/srv/data/zipline/uploads

5. Nginx 反向代理

站点文件：

  • /etc/nginx/sites-available/img.starryocean.qzz.io
  • /etc/nginx/sites-enabled/img.starryocean.qzz.io

代理链路：

  • img.starryocean.qzz.io:80/443
  • Nginx -> 127.0.0.1:3000

上传限制：

  • client_max_body_size 100M

6. HTTPS 证书

使用 certbot --nginx 签发：

  • 域名：img.starryocean.qzz.io

证书目录：

  • /etc/letsencrypt/live/img.starryocean.qzz.io/

到期时间：

  • 2026-08-06

7. 注册与访问控制收紧

7.1 实际策略

为了做到“只有当前人工已注册账号可用”，已直接在 Zipline 数据库层关闭以下能力：

  • 公开注册：关闭
  • OAuth 注册：关闭
  • 邀请码功能：关闭

7.2 已设置项

当前 Zipline 表中的关键值：

  • featuresUserRegistration = false
  • featuresOauthRegistration = false
  • invitesEnabled = false
  • coreReturnHttpsUrls = true
  • coreDefaultDomain = img.starryocean.qzz.io
  • domains = {img.starryocean.qzz.io}

7.3 说明

这意味着：

  • 新访客不能自己注册
  • 也不能通过邀请码再创建账号
  • 只有当前已经存在的账号可以登录使用

8. 运行验证

已验证：

  • http://127.0.0.1:3000/api/healthcheck 返回通过
  • https://img.starryocean.qzz.io/dashboard 可正常加载前端页面
  • 容器状态为 healthy

9. 关键维护命令

9.1 查看容器状态

docker ps

9.2 查看图床日志

docker logs -f zipline
docker logs -f zipline-postgres

9.3 重启图床

cd /srv/apps/zipline
docker compose up -d

9.4 重新加载 Nginx

nginx -t && systemctl reload nginx

10. 关键路径汇总

  • 编排文件：/srv/apps/zipline/compose.yaml
  • 环境变量：/srv/apps/zipline/.env
  • 上传目录：/srv/data/zipline/uploads
  • 数据库目录：/srv/data/zipline/postgres
  • Nginx 配置：/etc/nginx/sites-available/img.starryocean.qzz.io
  • 证书目录：/etc/letsencrypt/live/img.starryocean.qzz.io/

  • 如需接入客户端上传：
