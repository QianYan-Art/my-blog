================= 配置区域 (修改这里) =================

2025-12-185 min

适用场景：

  • 只有普通用户权限的 NAT VPS
  • 共享主机或容器环境（LXC/OpenVZ）
  • 管理员禁止使用 systemctl 的环境

核心原理：

  1. 运行：使用 nohup 后台运行二进制文件。
  1. 保活：利用 crontab 每分钟检查进程，挂了自动重启。
  1. 日志：配置自动轮转，防止日志写满磁盘。

一、 极速部署脚本（推荐）

为了避免使用 vi/nano 编辑器导致的格式错误，请直接修改下方的变量区域，然后复制整段代码到终端执行。

1. 准备配置参数

请根据你的实际情况，修改下面前三行内容：

# ================= 配置区域 (修改这里) =================
# 探针下载地址
DOWNLOAD_URL="[https://github.com/komari-monitor/komari-agent/releases/latest/download/komari-agent-linux-amd64](https://github.com/komari-monitor/komari-agent/releases/latest/download/komari-agent-linux-amd64)"
# 你的面板地址和 Token (注意保留引号)
START_ARGS="-e [https://komari.starryocean.qzz.io](https://komari.starryocean.qzz.io) -t AQhDXdx50jMUgkqQ1SipPw"
# 进程名称 (用于检测是否存活，通常是二进制文件的名字)
PROCESS_NAME="komari-agent"
# ======================================================

# ================= 下面不要动，直接复制 =================
# 1. 准备目录
WORK_DIR="$HOME/$PROCESS_NAME"
mkdir -p "$WORK_DIR"
cd "$WORK_DIR"

# 2. 下载并授权
echo "正在下载探针..."
curl -L -o "$PROCESS_NAME" "$DOWNLOAD_URL"
chmod +x "$PROCESS_NAME"

# 3. 生成保活脚本 (使用 EOF 写入，避免编辑器问题)
cat > "$WORK_DIR/keepalive.sh" << EOF
#!/bin/bash
if ! pgrep -f "$PROCESS_NAME" > /dev/null; then
    cd "$WORK_DIR"
    nohup ./$PROCESS_NAME $START_ARGS > nohup.out 2>&1 &
    echo "Restarted at \$(date)" >> "$WORK_DIR/restart.log"
fi
EOF
chmod +x "$WORK_DIR/keepalive.sh"

# 4. 生成日志轮转配置 (防止磁盘爆满)
mkdir -p $HOME/.config/logrotate
cat > $HOME/.config/logrotate/$PROCESS_NAME << EOF
$WORK_DIR/nohup.out {
    daily
    rotate 3
    compress
    missingok
    notifempty
    copytruncate
    maxsize 5M
}
EOF

# 5. 写入 Crontab 定时任务 (暴力覆盖模式，确保成功)
# 注意：这会覆盖当前用户的其他定时任务。如有其他任务请手动添加。
echo "0 2 * * * /usr/sbin/logrotate -s $HOME/.config/logrotate/status $HOME/.config/logrotate/$PROCESS_NAME
* * * * * $WORK_DIR/keepalive.sh" | crontab -

# 6. 立即启动
$WORK_DIR/keepalive.sh

echo "----------------------------------------"
echo "部署完成！"
echo "请等待 1 分钟后检查面板状态。"
echo "检查命令: ps aux | grep $PROCESS_NAME"
echo "----------------------------------------"

二、 常用维护命令

1. 检查探针是否在运行

ps aux | grep komari-agent

_如果有输出说明正常。_

2. 查看当前定时任务

crontab -l

_必须看到 * ... keepalive.sh 这一行才算拥有“复活甲”。_

3. 查看崩溃/重启记录

cat ~/komari-agent/restart.log

_如果文件不存在或为空，说明非常稳定。如果有很多时间点，说明经常崩溃（但被自动救回来了）。_

4. 彻底卸载

如果这台机器不想要了，执行以下命令清理干净：

# 1. 清空定时任务
crontab -r

# 2. 杀掉进程
pkill -f komari-agent

# 3. 删除文件
rm -rf ~/komari-agent
rm -rf ~/.config/logrotate/komari-agent

三、 常见问题 (FAQ)

Q1: 为什么我用 crontab -e 修改后保存不了？

原因：默认编辑器通常是 vi，操作反直觉。

解决：

  1. 尽量使用本文提供的“全自动脚本”直接覆盖，不打开编辑器。
  1. 如果必须编辑，先执行 export EDITOR=nano，将编辑器切换为类似记事本的 nano 模式。

Q2: 脚本里的 2>&1 是什么意思？

这是将“错误输出”和“标准输出”合并。确保不管是正常的日志还是报错信息，都写入到 nohup.out 里，方便排查问题。

Q3: 这种方式会占用很多资源吗？

不会。crontab 是系统自带服务，keepalive.sh 脚本极其轻量，每分钟运行一次只消耗几乎可忽略的 CPU。

Q4: 适用于其他探针吗？

适用。比如哪吒探针（Nezha Agent），只需要将脚本开头的 DOWNLOAD_URL 换成哪吒的下载链接，START_ARGS 换成 -s 你的地址 -p 你的密钥 即可。
