🎯 环境要求

  • 操作系统：Ubuntu 22.04+ / Debian 11+
  • Python版本：3.10+
  • 权限：root用户或sudo权限
  • 阿里云账号：拥有ECS管理权限的AccessKey

🚀 部署步骤

步骤一：解决虚拟环境问题

如果提示ensurepip is not available，需先安装venv组件：

sudo apt update
sudo apt install -y python3.12-venv

步骤二：创建Python虚拟环境

# 删除可能存在的残留目录
rm -rf /root/aliyun-env

# 创建虚拟环境
python3 -m venv /root/aliyun-env

步骤三：安装阿里云SDK依赖

# 在虚拟环境中安装
/root/aliyun-env/bin/pip install aliyun-python-sdk-core aliyun-python-sdk-ecs

步骤四：创建监控脚本

在/root/目录下创建traffic_monitor.py文件：

cat > /root/traffic_monitor.py << 'EOF'
#!/root/aliyun-env/bin/python
# -*- coding: utf-8 -*-
from aliyunsdkcore.client import AcsClient
from aliyunsdkcore.request import CommonRequest
from aliyunsdkecs.request.v20140526 import StartInstancesRequest, StopInstancesRequest, DescribeInstancesRequest
import json
import sys
import logging

# ================== 配置区域 ==================
ACCESS_KEY_ID = 'LTAIxxxxxxx'      # ⚠️ 替换为您的AccessKey ID
ACCESS_KEY_SECRET = 'xxxxxxxxxx'   # ⚠️ 替换为您的AccessKey Secret
REGION_ID = 'cn-hongkong'          # 区域ID
ECS_INSTANCE_ID = 'i-xxxxxxxxx'    # ⚠️ 替换为您的ECS实例ID
TRAFFIC_THRESHOLD_GB = 180        # 流量阈值(GB)

# ================== 日志配置 ==================
logging.basicConfig(
    level=logging.WARNING,         # 日志级别：WARNING/INFO/DEBUG
    format="%(asctime)s - %(levelname)s - %(message)s",
    stream=sys.stdout
)
logger = logging.getLogger(__name__)

# ================== 初始化客户端 ==================
try:
    client = AcsClient(ACCESS_KEY_ID, ACCESS_KEY_SECRET, REGION_ID)
    logger.warning("AcsClient初始化成功")
except Exception as e:
    logger.error(f"AcsClient初始化失败: {e}")
    sys.exit(1)

# ================== 功能函数 ==================
def get_total_traffic_gb(client):
    """查询当前总互联网流量"""
    request = CommonRequest()
    request.set_domain('cdt.aliyuncs.com')
    request.set_version('2021-08-13')
    request.set_action_name('ListCdtInternetTraffic')
    request.set_method('POST')
    try:
        response = client.do_action_with_exception(request)
        response_json = json.loads(response.decode('utf-8'))
        total_bytes = sum(d.get('Traffic', 0) for d in response_json.get('TrafficDetails', []))
        total_gb = total_bytes / (1024 ** 3)
        logger.warning(f"当前总互联网流量: {total_gb:.2f} GB")
        return total_gb
    except Exception as e:
        logger.error(f"获取CDT流量失败: {e}")
        sys.exit(1)

def get_ecs_status(client, instance_id):
    """查询ECS实例状态"""
    try:
        request = DescribeInstancesRequest.DescribeInstancesRequest()
        request.set_InstanceIds([instance_id])
        response = client.do_action_with_exception(request)
        response_json = json.loads(response.decode('utf-8'))
        instances = response_json.get("Instances", {}).get("Instance", [])
        if not instances:
            logger.error("未找到该ECS实例信息")
            return None
        status = instances[0].get("Status")
        logger.warning(f"ECS实例 {instance_id} 当前状态: {status}")
        return status
    except Exception as e:
        logger.error(f"获取ECS实例状态失败: {e}")
        return None

def ecs_start(client, instance_id):
    """启动ECS实例"""
    status = get_ecs_status(client, instance_id)
    if status == "Running":
        logger.warning(f"ECS实例 {instance_id} 已经是运行状态")
        return
    try:
        request = StartInstancesRequest.StartInstancesRequest()
        request.set_InstanceIds([instance_id])
        request.set_accept_format('json')
        response = client.do_action_with_exception(request)
        logger.warning(f"ECS启动响应: {response.decode('utf-8')}")
    except Exception as e:
        logger.error(f"启动ECS实例失败: {e}")

def ecs_stop(client, instance_id):
    """停止ECS实例"""
    status = get_ecs_status(client, instance_id)
    if status == "Stopped":
        logger.warning(f"ECS实例 {instance_id} 已经是停止状态")
        return
    try:
        request = StopInstancesRequest.StopInstancesRequest()
        request.set_InstanceIds([instance_id])
        request.set_ForceStop(False)
        request.set_accept_format('json')
        response = client.do_action_with_exception(request)
        logger.warning(f"ECS停止响应: {response.decode('utf-8')}")
    except Exception as e:
        logger.error(f"停止ECS实例失败: {e}")

def main():
    """主流程"""
    total_gb = get_total_traffic_gb(client)
    if total_gb < TRAFFIC_THRESHOLD_GB:
        logger.warning(f"流量 {total_gb:.2f} GB < 阈值 {TRAFFIC_THRESHOLD_GB} GB，启动ECS")
        ecs_start(client, ECS_INSTANCE_ID)
    else:
        logger.warning(f"流量 {total_gb:.2f} GB ≥ 阈值 {TRAFFIC_THRESHOLD_GB} GB，停止ECS")
    logger.warning("脚本执行完毕")

if __name__ == "__main__":
    main()
EOF

⚠️ 重要：务必修改脚本中的以下参数：

  • ACCESS_KEY_ID
  • ACCESS_KEY_SECRET
  • ECS_INSTANCE_ID
  • TRAFFIC_THRESHOLD_GB（按需调整）

步骤五：配置定时任务（每分钟执行）

# 添加crontab任务
(crontab -l 2>/dev/null; echo "* * * * * /root/aliyun-env/bin/python /root/traffic_monitor.py >> /root/traffic_monitor.log 2>&1") | crontab -

# 查看已配置的定时任务
crontab -l

📊 日志管理配置

方案：使用logrotate自动管理日志

#### 1. 安装logrotate

sudo apt update
sudo apt install -y logrotate

#### 2. 创建logrotate配置文件

sudo tee /etc/logrotate.d/traffic_monitor > /dev/null << 'EOF'
/root/traffic_monitor.log {
    daily                    # 每天轮转一次
    rotate 7                 # 保留7个旧日志
    compress                 # 压缩旧日志
    delaycompress            # 延迟压缩，保留一个未压缩
    missingok                # 日志不存在时不报错
    notifempty               # 日志为空时不轮转
    create 644 root root     # 创建新日志的权限
}
EOF

#### 3. 调整日志级别（减少日志量）

# 将日志级别从INFO改为WARNING
sed -i 's/level=logging.INFO/level=logging.WARNING/' /root/traffic_monitor.py

#### 4. 重启cron服务

systemctl restart cron

🔧 故障排查

问题1：虚拟环境创建失败

现象：ensurepip is not available 解决：

sudo apt install -y python3.12-venv
rm -rf /root/aliyun-env
python3 -m venv /root/aliyun-env

问题2：脚本执行无输出

检查项：

# 检查脚本权限
ls -l /root/traffic_monitor.py

# 检查虚拟环境Python路径
ls -l /root/aliyun-env/bin/python

# 手动执行测试
/root/aliyun-env/bin/python /root/traffic_monitor.py

问题3：定时任务未执行

检查项：

⚠️ 注意事项

  1. 安全凭证管理

    2. - 建议将AccessKey存储在环境变量或配置文件中，而非硬编码 - 示例：export ALIYUN_ACCESS_KEY_ID="your_key"，然后在脚本中读取 - 定期轮换AccessKey，避免泄露风险

  1. 日志级别选择

    2. - logging.WARNING：生产环境推荐，日志量少 - logging.INFO：调试阶段使用，可查看详细流程 - logging.DEBUG：开发环境使用，输出最详细信息

  1. 流量阈值设置

    2. - 建议设置低于阿里云CDT套餐阈值（如套餐200GB，设置180GB） - 给脚本留出足够的反应时间（每分钟检查一次）

  1. ECS实例状态

    2. - 脚本会自动判断实例当前状态，避免重复操作 - 停止实例使用ForceStop=False，允许优雅关机

  1. 监控与告警

    2. - 建议添加钉钉/邮件告警功能，当触发启停时通知 - 示例：在ecs_start和ecs_stop函数中调用Webhook

  1. 成本优化

    2. - 停止ECS实例后，仅收取磁盘费用 - 检查ECS是否使用按量付费模式

  1. 日志文件路径

    2. - 所有文件使用绝对路径，避免cron环境变量问题 - 确保/root/目录有足够磁盘空间

  1. 权限管理

    2. - 当前使用root用户部署，生产环境建议创建专用用户 - 确保AccessKey权限最小化原则（仅授权ECS和CDT只读权限）
