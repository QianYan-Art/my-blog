1. 环境准备与 Hello World

安装

# 官方推荐方式（所有平台通用）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 验证
rustc --version   # 编译器版本
cargo --version   # 包管理器版本

第一个程序

文件名：main.rs

fn main() {
    println!("Hello, world!");
}

编译运行：

rustc main.rs   # 编译
./main          # 运行（Windows 为 main.exe）

对比 TS/JS：Rust 是编译型语言，需要先编译再执行，没有解释器直接运行。

2. 变量与可变性

不可变变量（默认）

let x = 5;
// x = 6;  ← 编译错误！默认不可变
println!("x = {}", x);

对比 Python/JS：类似 Python 的常量约定，但 Rust 在编译期强制检查，不是运行时。

可变变量

let mut x = 5;
x = 6;  // OK
println!("x = {}", x);

mut 关键字显式声明可变性。Rust 鼓励默认不可变，减少意外修改。

常量

const MAX_POINTS: u32 = 100_000;  // 必须标注类型
  • const 永远不可变，不能用 mut
  • 必须显式标注类型
  • 可声明在任何作用域（包括全局）
  • 只能绑定到常量表达式，不能是函数调用结果

对比 Java：类似 static final，但 Rust 常量更严格，编译期求值。

变量遮蔽（Shadowing）

let x = 5;
let x = x + 1;      // 新变量遮蔽旧变量
let x = x * 2;
println!("x = {}", x);  // 12

// 遮蔽可以改变类型
let spaces = "   ";     // &str
let spaces = spaces.len(); // usize

对比 JS let：JS 中 let 重新赋值会报错（同一作用域），Rust 的遮蔽是创建新变量，旧变量不再可访问。

3. 数据类型

Rust 是静态类型语言，编译器必须在编译期知道每个变量的类型。

标量类型（Scalar Types）

整数
类型 范围 示例
i8 -128 ~ 127 let a: i8 = -42;
u8 0 ~ 255 let a: u8 = 255;
i32 -2³¹ ~ 2³¹-1 默认整数类型
u32 0 ~ 2³²-1 let a: u32 = 100;
i64 -2⁶³ ~ 2⁶³-1 let a: i64 = 1_000_000;
i128 / u128 128位 大整数运算
isize / usize 指针大小 集合索引、长度 
let decimal = 98_222;       // 十进制，下划线分隔提高可读性
let hex = 0xff;              // 十六进制
let octal = 0o77;            // 八进制
let binary = 0b1111_0000;    // 二进制
let byte = b'A';             // 字节字面量（u8）

对比 Java：类型名更短（i32 vs int），且没有隐式窄化转换。

浮点数
let x = 2.0;      // f64（默认）
let y: f32 = 3.0; // f32
布尔值
let t = true;
let f: bool = false;
字符
let c = 'z';        // char，4 字节 Unicode 标量值
let heart = '💖';   // 支持 emoji

对比 JS/Python：Rust 的 char 是 4 字节 Unicode 标量，不是 UTF-8 字节。

复合类型（Compound Types）

元组（Tuple）
let tup: (i32, f64, u8) = (500, 6.4, 1);

// 解构（destructuring）
let (x, y, z) = tup;
println!("y = {}", y);  // 6.4

// 索引访问
let five_hundred = tup.0;
let six_point_four = tup.1;

对比 TS：类似 [number, string] 元组类型，但 Rust 元组是匿名的，没有命名字段。

数组（Array）
let a = [1, 2, 3, 4, 5];          // 固定长度，同类型
let first = a[0];

// 显式标注类型
let a: [i32; 5] = [1, 2, 3, 4, 5];  // [类型; 长度]

// 初始化所有元素为同一值
let a = [3; 5];  // [3, 3, 3, 3, 3]

对比 JS 数组：Rust 数组长度固定、类型统一，更像 C/Java 数组。动态数组用 Vec<T>。

类型推断与标注

let guess: u32 = "42".parse().expect("不是数字！");
// 如果不标注类型，编译器无法推断 parse() 要转成什么类型

4. 函数

基本定义

fn main() {
    println!("Hello");
    another_function(5, 'h');
}

fn another_function(x: i32, unit_label: char) {
    println!("值为 {x} {unit_label}");
}
  • fn 关键字声明函数
  • 参数必须标注类型
  • 函数名用 snake_case

语句与表达式

Rust 是表达式语言，理解语句和表达式的区别至关重要：

// 语句（statement）：执行操作，不返回值
let y = 6;  // 语句

// 表达式（expression）：计算并返回值
let x = 5 + 3;  // 5 + 3 是表达式，返回 8

// 代码块也是表达式
let y = {
    let x = 3;
    x + 1  // 注意：没有分号！这是表达式，返回 4
};
println!("y = {}", y);  // 4

对比 JS：JS 中 { x + 1 } 是代码块，不返回值。Rust 中代码块是表达式，最后一行（无分号）的值就是返回值。

返回值

fn five() -> i32 {
    5  // 无分号，作为表达式返回
}

fn plus_one(x: i32) -> i32 {
    x + 1  // 表达式，自动返回
}

// 注意：加一分号就变成语句，编译错误！
// fn bad() -> i32 {
//     x + 1;  // 错误：() 不匹配 i32
// }

对比 Python：不需要 return 关键字，最后一行表达式自动返回（类似 Ruby）。但 Rust 的返回值类型在签名中显式声明。

5. 控制流

if 表达式

let number = 6;

if number % 2 == 0 {
    println!("偶数");
} else {
    println!("奇数");
}

// if 是表达式，可以赋值给变量
let condition = true;
let number = if condition { 5 } else { 6 };
// let number = if condition { 5 } else { "six" }; // 错误：类型必须一致！

对比 JS 三元运算符：let x = cond ? a : b，但 Rust 的 if 更强大，支持代码块。

循环

loop — 无限循环
let mut counter = 0;
let result = loop {
    counter += 1;
    if counter == 10 {
        break counter * 2;  // break 可以带返回值！
    }
};
println!("result = {}", result);  // 20
while — 条件循环
let mut number = 3;
while number != 0 {
    println!("{number}!");
    number -= 1;
}
for — 遍历循环
let a = [10, 20, 30, 40, 50];

// 遍历集合
for element in a {
    println!("值: {element}");
}

// 遍历范围
for number in 1..4 {    // 1, 2, 3（不包含 4）
    println!("{number}");
}

// 反向遍历
for number in (1..4).rev() {
    println!("{number}!");
}
// 输出: 3! 2! 1!

对比 Python：for x in range(1, 4) ≈ for x in 1..4。Rust 的 .. 是半开区间。

模式匹配补充：if let

let some_value: Option<i32> = Some(5);

// 只想匹配 Some，不关心 None
if let Some(x) = some_value {
    println!("值为 {x}");
}

6. 所有权系统（Ownership）

所有权是 Rust 最核心的特性，也是与其他语言最大的区别。

三条规则

  1. 每个值都有一个所有者（owner）
  2. 同一时间只有一个所有者
  3. 所有者离开作用域，值被丢弃（drop）

所有权转移（Move）

let s1 = String::from("hello");
let s2 = s1;  // s1 的所有权转移给 s2
// println!("{}", s1);  // 错误！s1 不再有效
println!("{}", s2);     // OK

对比 Python/JS：赋值是浅拷贝（引用+1）。Rust 的 String 赋值是移动语义——s1 失效，避免双重释放。

克隆（Clone）

let s1 = String::from("hello");
let s2 = s1.clone();  // 深拷贝
println!("s1 = {s1}, s2 = {s2}");  // 两者都有效

栈类型：复制语义（Copy Trait）

let x = 5;
let y = x;  // 整数实现了 Copy trait，赋值是复制
println!("x = {x}, y = {y}");  // 两者都有效

实现了 Copy trait 的类型：所有整数、浮点、布尔、字符、以及元素都实现 Copy 的元组。

函数中的所有权

fn main() {
    let s = String::from("hello");
    takes_ownership(s);    // s 被移动进函数
    // println!("{}", s);  // 错误！s 已失效

    let x = 5;
    makes_copy(x);         // i32 是 Copy，x 仍然有效
    println!("x = {x}");   // OK
}

fn takes_ownership(s: String) {
    println!("{s}");
}  // s 离开作用域，drop

fn makes_copy(n: i32) {
    println!("{n}");
}  // n 离开作用域，无事发生

返回值的所有权

fn main() {
    let s1 = gives_ownership();       // 函数返回值转移给 s1
    let s2 = String::from("hello");
    let s3 = takes_and_gives_back(s2); // s2 移入，返回值移给 s3
}

fn gives_ownership() -> String {
    String::from("yours")
}

fn takes_and_gives_back(s: String) -> String {
    s  // 返回所有权
}

核心思想：Rust 通过所有权系统在编译期消除内存安全问题（悬垂指针、双重释放、内存泄漏），不需要 GC。

7. 引用与借用

引用（Reference）

fn main() {
    let s = String::from("hello");
    let len = calculate_length(&s);  // 借用，不转移所有权
    println!("'{s}' 的长度是 {len}");  // s 仍然有效！
}

fn calculate_length(s: &String) -> usize {
    s.len()
}  // s 离开作用域，但不拥有值，不 drop

&s 创建引用，函数签名 &String 表示借用。

可变引用

fn main() {
    let mut s = String::from("hello");
    change(&mut s);
    println!("{s}");  // "hello, world"
}

fn change(s: &mut String) {
    s.push_str(", world");
}

借用规则（极其重要）

// 规则 1：不可变引用可以有多个
let s = String::from("hello");
let r1 = &s;  // OK
let r2 = &s;  // OK
println!("{r1} {r2}");

// 规则 2：可变引用只能有一个
let mut s = String::from("hello");
let r1 = &mut s;
// let r2 = &mut s;  // 错误！不能同时有两个可变引用

// 规则 3：可变引用和不可变引用不能共存
let mut s = String::from("hello");
let r1 = &s;       // 不可变引用
// let r2 = &mut s;  // 错误！已有不可变引用，不能再创建可变引用

对比 Java：Java 的引用类似 Rust 的 &T（不可变借用），但 Rust 额外引入了 &mut T（可变借用）和编译期检查，杜绝数据竞争。

悬垂引用（Dangling Reference）

// fn dangle() -> &String {     // 编译错误！
//     let s = String::from("hello");
//     &s  // 返回局部变量的引用
// }  // s 被 drop，引用悬空

Rust 编译器禁止返回局部变量的引用。

8. 切片（Slice）

切片是对集合中连续部分的引用，不拥有数据。

fn main() {
    let s = String::from("hello world");

    let hello = &s[0..5];   // "hello"
    let world = &s[6..11];  // "world"

    // 语法糖
    let slice = &s[..5];     // 从开头
    let slice = &s[6..];     // 到结尾
    let slice = &s[..];      // 整个字符串
}

字符串字面量就是切片

let s = "Hello, world!";  // s 的类型是 &str

&str 是指向二进制文件中某点的不可变切片。

数组切片

let a = [1, 2, 3, 4, 5];
let slice = &a[1..3];  // &[2, 3]

实际应用

fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]
}

对比 Python：s[0:5] 创建新字符串（拷贝），Rust 的切片是零拷贝的视图。

9. 结构体（struct）

定义与实例化

struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}

fn main() {
    let user1 = User {
        active: true,
        username: String::from("zhangsan"),
        email: String::from("zhangsan@example.com"),
        sign_in_count: 1,
    };

    // 访问字段
    println!("{}", user1.username);

    // 修改字段（整个实例必须是 mut）
    let mut user2 = User { /* ... */ };
    user2.email = String::from("new@example.com");
}

对比 TS interface：TS 的 interface 只是类型描述，Rust struct 是真正的数据结构，有内存布局。

字段初始化简写

fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username,      // ≈ username: username
        email,         // ≈ email: email
        sign_in_count: 1,
    }
}

结构体更新语法（..）

let user2 = User {
    email: String::from("another@example.com"),
    ..user1  // 其余字段从 user1 获取
};
// 注意：user1 的 username 是 String（非 Copy），已被移动，user1.username 不再可用

元组结构体

struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

let black = Color(0, 0, 0);
let origin = Point(0, 0, 0);
// black 和 origin 是不同类型！

单元结构体

struct AlwaysEqual;  // 没有字段

let subject = AlwaysEqual;

用于实现 trait 而不需要存储数据。

方法（impl）

#[derive(Debug)]  // 自动派生 Debug trait
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    // 方法（有 self 参数）
    fn area(&self) -> u32 {
        self.width * self.height
    }

    // 方法可以修改自身
    fn scale(&mut self, factor: u32) {
        self.width *= factor;
        self.height *= factor;
    }

    // 关联函数（无 self，类似静态方法）
    fn square(size: u32) -> Self {
        Self {
            width: size,
            height: size,
        }
    }
}

fn main() {
    let mut rect = Rectangle { width: 30, height: 50 };
    println!("面积: {}", rect.area());

    rect.scale(2);
    println!("缩放后: {} x {}", rect.width, rect.height);

    let sq = Rectangle::square(10);  // 关联函数用 ::
    println!("正方形: {} x {}", sq.width, sq.height);
}

对比 Java class：impl 类似 class 的方法定义，但 Rust 将数据（struct）和行为（impl）分离，更灵活。

自动引用和解引用

rect.area();
// 编译器自动处理以下等价形式：
// (&rect).area()
// (&mut rect).scale(2)

10. 枚举（enum）与模式匹配

基本枚举

enum IpAddrKind {
    V4,
    V6,
}

let four = IpAddrKind::V4;
let six = IpAddrKind::V6;

带数据的枚举

enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
}

let home = IpAddr::V4(127, 0, 0, 1);
let loopback = IpAddr::V6(String::from("::1"));

对比 TS 联合类型：type IpAddr = { kind: "v4", a: number, ... } | { kind: "v6", addr: string }，但 Rust 枚举是真正的 tagged union，内存效率更高。

经典枚举：Option

enum Option<T> {
    None,
    Some(T),
}

let some_number = Some(5);
let absent_number: Option<i32> = None;

对比 JS：Rust 没有 null，用 Option<T> 显式表达"可能有值也可能没有"。编译器强制你处理 None 情况。

match 模式匹配

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => {
            println!("幸运 quarter!");
            25
        }
    }
}

match 必须穷尽所有可能性（exhaustive），编译器会检查。

匹配 Option

fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1),
    }
}

通配符 _

let dice_roll = 9;
match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    _ => reroll(),  // 匹配所有其他值
}

if let 简化匹配

let config_max = Some(3u8);

// 完整 match
match config_max {
    Some(max) => println!("最大值是 {max}"),
    _ => (),
}

// 简化写法
if let Some(max) = config_max {
    println!("最大值是 {max}");
}

while let

let mut stack = Vec::new();
stack.push(1);
stack.push(2);
stack.push(3);

while let Some(top) = stack.pop() {
    println!("{top}");
}
// 输出: 3, 2, 1

11. 泛型、Trait 与生命周期

泛型（Generics）

函数泛型
fn largest<T: PartialOrd + Copy>(list: &[T]) -> T {
    let mut largest = list[0];
    for &item in list {
        if item > largest {
            largest = item;
        }
    }
    largest
}
结构体泛型
struct Point<T, U> {
    x: T,
    y: U,
}

let integer_and_float = Point { x: 5, y: 4.0 };
枚举泛型
enum Result<T, E> {
    Ok(T),
    Err(E),
}

对比 Java 泛型：语法类似 <T>，但 Rust 泛型通过单态化（monomorphization）在编译期生成具体类型代码，零运行时开销。

Trait（特质）

Trait 定义共享行为，类似 Java 的 interface 或 TS 的 interface（但更强大）。

定义 trait
pub trait Summary {
    fn summarize(&self) -> String;

    // 默认实现
    fn default_summary(&self) -> String {
        String::from("(默认摘要)")
    }
}
实现 trait
pub struct NewsArticle {
    pub headline: String,
    pub author: String,
}

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{} by {}", self.headline, self.author)
    }
}
trait 作为参数
// 语法 1：impl Trait
pub fn notify(item: &impl Summary) {
    println!("快讯！{}", item.summarize());
}

// 语法 2：trait bound（更灵活，适合多参数）
pub fn notify2<T: Summary>(item: &T) {
    println!("快讯！{}", item.summarize());
}

// 多个 trait bound
pub fn notify3<T: Summary + Display>(item: &T) {}

// where 子句（复杂约束时更清晰）
pub fn some_function<T, U>(t: &T, u: &U) -> i32
where
    T: Display + Clone,
    U: Clone + Debug,
{
    // ...
}
返回 trait 类型
fn returns_summarizable() -> impl Summary {
    NewsArticle {
        headline: String::from("标题"),
        author: String::from("作者"),
    }
}
条件实现
impl<T: Display + PartialOrd> ToString for T {
    // ...
}

对比 TS interface：Rust trait 可以有默认实现，支持条件实现，还能为外部类型实现本地 trait（孤儿规则限制）。

生命周期（Lifetime）

生命周期确保引用有效，编译器通过生命周期标注推断引用的有效期。

基本标注
// 'a 表示：返回值的生命周期与两个参数中较短的那个一致
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
结构体中的生命周期
struct ImportantExcerpt<'a> {
    part: &'a str,  // 该引用的生命周期至少要和结构体实例一样长
}

fn main() {
    let novel = String::from("这是一个很长的故事...");
    let first_sentence = novel.split('.').next().unwrap();
    let excerpt = ImportantExcerpt {
        part: first_sentence,
    };
}
生命周期省略规则

编译器会自动推断大部分生命周期，三种规则：

  1. 每个引用参数获得一个独立生命周期
  2. 如果只有一个输入生命周期，所有输出引用继承它
  3. 如果有多个输入生命周期，但一个是 &self 或 &mut self，输出继承 self 的生命周期
// 编译器自动推断，等价于：
// fn first_word<'a>(s: &'a str) -> &'a str
fn first_word(s: &str) -> &str {
    // ...
}
静态生命周期
let s: &'static str = "我永远存在";  // 整个程序生命周期

为什么需要生命周期：Rust 没有 GC，需要确保引用不会指向已释放的内存。生命周期标注告诉编译器引用的有效期，编译期验证安全性。

12. 错误处理

Rust 没有异常（exception），用 Result 和 Option 显式处理错误。

panic! — 不可恢复错误

fn main() {
    panic!("崩溃！");  // 程序终止

    // 数组越界也会 panic
    let v = vec![1, 2, 3];
    v[99];  // panic: index out of bounds
}

Result — 可恢复错误

enum Result<T, E> {
    Ok(T),
    Err(E),
}
基本使用
use std::fs::File;

fn main() {
    let file = File::open("hello.txt");

    let file = match file {
        Ok(file) => file,
        Err(error) => panic!("打开文件失败: {error:?}"),
    };
}
传播错误：? 操作符
use std::fs::File;
use std::io::{self, Read};

fn read_username_from_file() -> Result<String, io::Error> {
    // ? 操作符：Ok 则解包，Err 则提前返回
    let mut file = File::open("hello.txt")?;
    let mut username = String::new();
    file.read_to_string(&mut username)?;
    Ok(username)
}

// 链式写法
fn read_username_from_file2() -> Result<String, io::Error> {
    let mut username = String::new();
    File::open("hello.txt")?.read_to_string(&mut username)?;
    Ok(username)
}

对比 Java try-catch：? 类似自动传播异常，但类型安全，不会有遗漏。

unwrap 和 expect
let file = File::open("hello.txt").unwrap();       // Ok 取值，Err 则 panic
let file = File::open("hello.txt").expect("文件必须存在"); // 自定义 panic 信息
自定义错误类型
use std::fmt;

#[derive(Debug)]
struct AppError {
    message: String,
}

impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "应用错误: {}", self.message)
    }
}

impl std::error::Error for AppError {}

fn do_something() -> Result<(), AppError> {
    Err(AppError {
        message: String::from("出错了"),
    })
}

对比 Python try/except：Rust 强制你处理错误，Result 和 Option 让错误处理成为类型系统的一部分。

13. 集合类型

Vector（动态数组）

let v: Vec<i32> = Vec::new();           // 空 vector
let v = vec![1, 2, 3];                  // 宏初始化

let mut v = Vec::new();
v.push(5);
v.push(6);

// 读取元素
let third: &i32 = &v[2];               // 索引访问，越界 panic
let third: Option<&i32> = v.get(2);    // get 返回 Option，越界返回 None

// 遍历
for i in &v {
    println!("{i}");
}

// 遍历并修改
for i in &mut v {
    *i += 50;  // 解引用
}

String

let mut s = String::from("hello");
s.push_str(" world");     // 追加字符串切片
s.push('!');              // 追加单个字符

let s1 = String::from("Hello, ");
let s2 = String::from("world!");
let s3 = s1 + &s2;       // s1 被移动，不再可用

// 格式化
let s = format!("{s2} {s3}");  // 不获取所有权

// 索引访问？不行！
// Rust String 不支持 s[0]，因为 UTF-8 变长编码

// 遍历字符
for c in "你好".chars() {
    println!("{c}");
}

// 遍历字节
for b in "你好".bytes() {
    println!("{b}");
}

对比 Python 字符串：Python 字符串支持索引 s[0]，Rust 不支持，因为 UTF-8 字符可能占 1-4 字节，索引操作是 O(n) 的。

HashMap

use std::collections::HashMap;

let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);
scores.insert(String::from("Yellow"), 50);

// 访问
let team = String::from("Blue");
let score = scores.get(&team);  // Option<&V>

// 遍历
for (key, value) in &scores {
    println!("{key}: {value}");
}

// 更新
scores.entry(String::from("Blue")).or_insert(50);  // 仅在不存在时插入

对比 Python dict：API 类似，但 Rust 的 HashMap 需要显式导入，且有所有权规则。

对比 ES Module：use 类似 import，pub 类似 export。Rust 的模块系统更严格，默认私有。

16. 并发基础

创建线程

use std::thread;
use std::time::Duration;

fn main() {
    let handle = thread::spawn(|| {
        for i in 1..10 {
            println!("子线程: {i}");
            thread::sleep(Duration::from_millis(1));
        }
    });

    for i in 1..5 {
        println!("主线程: {i}");
        thread::sleep(Duration::from_millis(1));
    }

    handle.join().unwrap();  // 等待子线程完成
}

移动语义与线程

let v = vec![1, 2, 3];

let handle = thread::spawn(move || {
    println!("{v:?}");
});

// println!("{v:?}");  // 错误！v 已被 move 到线程中

handle.join().unwrap();

move 闭包强制获取变量的所有权，确保线程安全。

消息传递（Channel）

use std::sync::mpsc;  // multiple producer, single consumer
use std::thread;

fn main() {
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let val = String::from("hello");
        tx.send(val).unwrap();
        // println!("{val}");  // 错误：val 已被 send 移动
    });

    let received = rx.recv().unwrap();
    println!("收到: {received}");
}

对比 Go channel：Rust 的 channel 类似 Go，但通过所有权系统保证发送后不能再使用已发送的值。

共享状态：Mutex

use std::sync::Mutex;

fn main() {
    let m = Mutex::new(5);

    {
        let mut num = m.lock().unwrap();
        *num = 6;
    }  // 锁自动释放

    println!("m = {m:?}");
}

Arc（原子引用计数）

use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("结果: {}", *counter.lock().unwrap());
}

对比 Java 并发：Rust 的类型系统防止数据竞争——Mutex<T> 和 Arc<T> 的组合确保线程安全在编译期验证。

18. 综合示例

命令行猜数字游戏

use rand::Rng;
use std::cmp::Ordering;
use std::io;

fn main() {
    println!("猜数字游戏！");

    let secret_number = rand::thread_rng().gen_range(1..=100);

    loop {
        println!("请输入你的猜测：");

        let mut guess = String::new();
        io::stdin()
            .read_line(&mut guess)
            .expect("读取失败");

        let guess: u32 = match guess.trim().parse() {
            Ok(num) => num,
            Err(_) => continue,
        };

        println!("你猜的是: {guess}");

        match guess.cmp(&secret_number) {
            Ordering::Less => println!("太小了！"),
            Ordering::Greater => println!("太大了！"),
            Ordering::Equal => {
                println!("猜对了！");
                break;
            }
        }
    }
}
