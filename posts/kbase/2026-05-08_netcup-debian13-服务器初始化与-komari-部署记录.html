6.8 主题默认视图修正

升级到 1.2.0 后，前端“表格视图 / 紧凑视图 / 网格视图”存在浏览器本地存储不稳定的问题。

现象：

  • 在某些浏览器 Profile 中，手动切换为 table 后，重新打开页面又恢复成 grid
  • 不同 Chrome 账号在同一台电脑上的表现不一致

排查结论：

  • 主题配置中启用了 enableLocalStorage = true
  • 前端优先尝试使用浏览器本地存储记住视图
  • 某些浏览器 Profile 未能稳定保留该项
  • 因此仅靠浏览器侧记忆不可靠

最终处理：

  • 直接修改 Komari 主题数据库默认值
  • 将桌面端默认视图改为 table
  • 将移动端默认视图设为 compact

当前结果：

  • 即使浏览器本地存储失效，默认也不会再退回 grid

6.9 favicon 与浏览器标签图标修正

后续对面板站点又做了一次 favicon 修复。

#### 问题现象

  • 面板后台上传 favicon 后，部分浏览器仍显示旧图标
  • 同一台机器上，不同 Chrome 账号表现不同
  • 无痕模式不再显示旧图标，但会显示空白图标

#### 排查结论

根因分两层：

  1. 浏览器 Profile 级 favicon 缓存顽固存在
  2. 上传的 .ico 文件本身不适合作为 Chrome 标签页 favicon

具体表现：

  • 原始 favicon.ico 只有单帧
  • 仅包含 512x512
  • 对 Chrome 标签页不够友好

#### 实际修复

  1. 服务器继续保留：

    2. - /opt/komari/data/favicon.ico

  2. 新增更稳的标签图标文件：

    3. - /opt/komari/data/favicon-32.png

  3. 修改当前主题静态入口文件：

    4. - /opt/komari/data/theme/PurCarte/dist/index.html

  4. 页面头部改为优先使用：

    5. - rel="icon" type="image/png" sizes="32x32" href="/favicon-32.png?v=20260509b"

  5. 同时保留：

    6. - rel="shortcut icon" 指向 .ico - rel="apple-touch-icon" 指向 .png

#### 最终结果

  • 浏览器标签图标已恢复正常显示
  • 通过新路径与版本号参数绕开旧缓存
  • 主题页与登录页均可正常引用新图标
