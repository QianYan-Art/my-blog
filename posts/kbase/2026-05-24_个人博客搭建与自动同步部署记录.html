个人博客搭建与自动同步部署记录

最后更新：2026-05-24

1. 目标

本次记录用于归档 www.shiqianyan.cn 个人博客从本地静态开发到线上部署、再到自动同步知识库文章的完整过程，重点覆盖：

  1. 博客当前采用的技术形态。
  2. 腾讯云服务器上的实际部署目录与访问方式。
  3. 私有知识库文章如何生成、如何上线。
  4. GitHub 直连不稳定时的备用同步方案。

2. 当前架构结论

当前博客不是数据库驱动站点，也不是 SSR 应用，而是：

  • 一个纯静态博客前端项目。
  • 首页、文章索引页、文章详情页最终都会生成成静态 html。
  • 线上由 nginx 直接提供静态文件。

实际分工如下：

模块作用
D:\Answer\blog博客前端与静态生成脚本项目
D:\Answer\QianYan-KBase\public公开文章源目录
scripts/sync-kbase.js将 Markdown 文章生成 articles.json 和静态文章页
/www/wwwroot/blog腾讯云服务器上的线上站点目录
nginx对外提供 www.shiqianyan.cn 的静态访问

3. 文章来源规则

博客只展示知识库仓库中的公开文章，即：

  • 只读取 QianYan-KBase/public
  • 不读取 QianYan-KBase/private

当前公开文章进一步按目录分组：

目录博客中的分类
public/my_local本地记录
public/my_server服务器记录

这意味着博客展示的每一篇文章，本质上都来自 public 下的 Markdown 文件，再由同步脚本生成静态结果。

4. 文章生成机制

同步脚本位置：

  • D:\Answer\blog\scripts\sync-kbase.js

生成结果有两类：

  1. assets/data/articles.json

    2. - 用于文章列表页读取索引 - 包含标题、摘要、发布日期、分类、标签、跳转地址等元信息

  1. posts/kbase/*.html

    2. - 每篇文章各自对应一个静态详情页 - 浏览器点击文章卡片后，实际打开的就是这里生成出来的静态 HTML

也就是说：

  • 服务器上会存文章
  • 但存的不是原始 Markdown
  • 而是同步脚本生成后的静态 HTML 和索引 JSON

5. 文章详情页是怎么打开的

文章列表页本身不会直接解析 Markdown。

当前浏览逻辑是：

  1. 浏览器打开 /blog/
  2. 前端读取 /assets/data/articles.json
  3. 渲染文章卡片
  4. 点击某篇文章后，跳转到对应的 /posts/kbase/xxx.html

因此文章详情页的来源不是“前端临时渲染 Markdown”，而是：

  • 同步阶段提前生成好的静态详情页

这个结构的优点是：

  • 页面打开快
  • 不暴露 GitHub token
  • 不需要线上浏览器再去读私库
  • nginx 直接托管即可

旧站在本次替换前已做过备份。

7. GitHub 自动同步方案

由于文章来源是私有知识库，后续采用“服务器定时同步 GitHub 私库文章”的方式更新博客内容。

当前服务器端同步脚本：

  • /usr/local/bin/blog-sync-kbase.sh

当前环境变量文件：

  • /etc/blog-sync.env

当前计划任务：

  • 每 15 分钟执行一次同步

同步流程如下：

  1. 服务器读取 /etc/blog-sync.env
  2. 使用 GITHUB_TOKEN 访问 QianYan-KBase
  3. 只同步 public 目录下的 Markdown
  4. 重新生成：

    5. - assets/data/articles.json - posts/kbase/*.html

  5. nginx 继续直接服务这些更新后的静态文件

8. GitHub 连通性与代理兜底

这台腾讯云服务器对 GitHub 的连通性并不完全稳定。

实际测试结果是：

  • api.github.com 可以访问
  • 但直连 github.com / git ls-remote 曾出现超时

因此当前同步策略做成了两层：

  1. 先直连 GitHub API 同步
  2. 如果直连失败，再通过境外腾讯云机器做临时代理重试

备用代理来源：

  • 124.156.206.155:23333

这台境外腾讯云机器上已部署 sing-box，作为兜底通路使用。

结论：

  • 平时优先直连，结构更简单
  • 直连抖动时，脚本会尝试借助境外机继续完成文章同步

9. 本次博客项目修正项

本次围绕博客本身还做了这些修正：

  1. 全站 favicon 改为新的个人图标资源。
  2. 文章索引页按发布日期倒序显示。
  3. 文章右上角日期改为实际发布日期，不再显示 Undated。
  4. Markdown 同步生成时补充了更完整的日期来源处理。
  5. 同步脚本改成“先生成到临时目录，再整体替换”，避免同步失败把线上文章清空。

10. 后续维护规则

以后新增或修改文章时，推荐遵守以下规则：

  1. 只把需要公开展示的文章放进 QianYan-KBase/public
  2. 本地文章放 public/my_local
  3. 服务器运维文章放 public/my_server
  4. 文件名尽量带日期前缀，如：

    5. - 2026-05-24_主题.md

  5. 有需要时可在 Front Matter 中显式写：

    6. - title - date - summary - tags

这样博客侧的排序、摘要和展示都会更稳定。

11. 当前结论

截至本次记录，博客已经具备以下能力：

  • 首页与文章页静态展示完成
  • 私有知识库公开文章可同步到博客
  • 线上站点已部署到腾讯云服务器
  • 文章详情页使用静态生成结果
  • 服务器已具备定时同步能力
  • GitHub 直连不稳时有境外代理兜底

后续如果只是在 QianYan-KBase/public 中新增文章，并正常推送到 GitHub，博客就可以继续沿用当前链路自动更新。
