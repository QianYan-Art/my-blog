3.3 打包流程

第一次打包（错误）：使用 Jetbrains 全局 Python 3.12.8 打包，导致 ttkbootstrap 和 pdfminer.six 未被包含进 exe。

第二次打包（正确）：使用 uv 管理的 .venv（Python 3.13.4）打包：

uv add --dev pyinstaller
uv run pyinstaller --noconfirm --onedir --windowed --name "LM-Translate-GUI" \
  --add-data "gui_prefs.template.json;." \
  --hidden-import "ttkbootstrap" --collect-all ttkbootstrap \
  --hidden-import "pdfminer" --collect-all pdfminer \
  gui.py

产物：

  • dist/LM-Translate-GUI/LM-Translate-GUI.exe（13 MB）
  • zip/LM-Translate-GUI-v1.3.1.zip（61 MB）

6. 本次操作中暴露的问题与教训

6.1 打包环境不一致

问题：首次打包用了 Jetbrains 全局 Python，而非项目 uv 环境，导致依赖缺失。

教训：打包时必须用 uv run pyinstaller ...，不能用全局 Python。

6.2 .gitignore 被误删

问题：rebase 时同步了远端删除 .gitignore 的提交，未及时发现。

教训：rebase 后应检查变更内容，关键文件（.gitignore、pyproject.toml 等）被修改时要确认原因。

6.3 二进制产物被提交

问题：.gitgitignore 缺失期间，dist/ 下 1272 个二进制文件被提交。

教训：.gitignore 应在项目初始化时就位，不应依赖事后补救。

6.4 旧压缩包被误删

问题：清理 zip 目录时删除了旧版本压缩包（translate_exe_v1.0.0.zip ~ v1.3.0.zip），这些文件从未被 git 追踪，无法恢复。

教训：清理操作前应确认文件是否被 git 追踪，未被追踪的文件删除前需谨慎。
