哪吒监控 (Nezha Monitoring) 完整部署与管理指南

2025-11-206 min
c-仅记录

适用系统： Ubuntu 22.04 LTS

本教程将指导你完成哪吒监控面板的搭建、被控端（Agent）的接入，以及如何彻底删除它们。

📋 第一部分：准备工作

在开始之前，请确保你拥有：

  1. 两台或以上服务器（一台做面板，其余做被监控端）。

  2. 一个域名（解析到面板服务器 IP）。

  3. GitHub 账号（用于 OAuth 登录后台）。

1. 获取 GitHub OAuth Client ID (面板端必需)

哪吒面板默认使用 GitHub 账号登录，需要先申请授权：

  1. 访问 GitHub Developer Settings。

  2. 点击 "New OAuth App"。

  3. 填写信息：

    • Application name: 随意填（如 Nezha Monitor）。

    • Homepage URL: http://你的域名:8008 (如果是通过域名访问且开了SSL，则是 https://...)。

    • Authorization callback URL: http://你的域名:8008/oauth2/callback (注意：如果是 HTTPS 请改为 https)。

  4. 点击注册，记录下 Client ID 和 Client Secret。

🖥️ 第二部分：安装面板端 (Dashboard)

登录你的主控服务器（安装面板的那台），执行以下步骤。

1. 运行官方安装脚本

哪吒官方脚本集成了 Docker 安装、面板配置等功能。

curl -L [https://raw.githubusercontent.com/nezhahq/scripts/main/install.sh](https://raw.githubusercontent.com/nezhahq/scripts/main/install.sh) -o nezha.sh && chmod +x nezha.sh && ./nezha.sh

如果是国内服务器无法连接 GitHub，可以使用国内镜像（需自行寻找可信源）或配置代理。

2. 配置安装选项

运行脚本后，选择 1. 安装面板端，脚本会引导你进行配置：

  1. Docker 安装：如果系统未安装 Docker，脚本会询问是否安装，输入 y 确认。

  2. 配置信息：

    • OAuth 提供商：选择 GitHub。

    • Client ID：输入上面获取的 GitHub Client ID。

    • Client Secret：输入上面获取的 GitHub Client Secret。

    • 用户名：输入你的 GitHub 用户名（作为超级管理员）。

    • 端口设置：默认访问端口 8008，Agent通信端口 5555（保持默认即可，除非端口被占用）。

安装完成后，访问 http://你的域名:8008 即可进入后台。

3. 配置反向代理与 SSL (可选但推荐)

为了安全，建议使用 Nginx 进行反向代理并配置 HTTPS。

🔌 第三部分：安装被控端 (Agent)

登录你的被监控服务器（客户端），或者在面板服务器本机安装（如果也要监控面板机）。

1. 在面板中添加服务器

  1. 登录哪吒面板后台。

  2. 点击菜单栏的 “主机” (Server)。

  3. 点击 “添加服务器” (Add Server)。

    • 名称：给服务器起个名（如 US-Node-01）。

    • 分组：可选，留空即可。

  4. 点击 “添加”。

2. 获取一键安装命令

添加成功后，你会看到一个小图标（通常是回形针或Linux图标），点击它，复制 “Linux 一键安装” 命令。

命令格式通常如下（请使用你面板生成的实际命令）：

curl -L [https://raw.githubusercontent.com/nezhahq/scripts/main/agent/install.sh](https://raw.githubusercontent.com/nezhahq/scripts/main/agent/install.sh) -o agent.sh && chmod +x agent.sh && env NZ_SERVER=你的面板域名:5555 NZ_TLS=true NZ_CLIENT_SECRET=你的密钥 ./agent.sh

3. 在被控端执行

在被监控的 Ubuntu 22.04 服务器上粘贴并运行上述命令。

4. 常用管理命令 (被控端)

Ubuntu 使用 Systemd 管理服务，以下是常用命令：

  • 启动 Agent：

    sudo systemctl start nezha-agent

  • 停止 Agent：

    sudo systemctl stop nezha-agent

  • 重启 Agent：

    sudo systemctl restart nezha-agent

  • 查看日志 (排查离线问题)：

    sudo journalctl -u nezha-agent -f

🗑️ 第四部分：删除与卸载

如果不需要哪吒面板了，请按照以下步骤彻底清除。

1. 卸载被控端 (Agent)

在被监控的服务器上执行：

方法 A：使用脚本卸载（推荐）

重新运行安装脚本，通常会有一个“卸载 Agent”的选项。

./agent.sh
# 如果脚本删了，可以重新下载下来运行，然后选择卸载选项

方法 B：手动彻底删除

# 1. 停止服务
sudo systemctl stop nezha-agent
# 2. 禁止开机自启
sudo systemctl disable nezha-agent
# 3. 删除二进制文件
sudo rm -rf /opt/nezha
# 4. 删除 Systemd 服务文件
sudo rm /etc/systemd/system/nezha-agent.service
# 5. 重载系统配置
sudo systemctl daemon-reload

2. 卸载面板端 (Dashboard)

在面板服务器上执行：

方法 A：使用脚本卸载

./nezha.sh
# 选择 "卸载面板" 选项

方法 B：手动彻底删除 (Docker 方式)

# 1. 停止并删除容器
docker stop nezha-dashboard
docker rm nezha-dashboard

# 2. 删除数据卷 (警告：这将删除所有面板数据)
docker volume rm nezha-dashboard-data

# 3. (可选) 删除拉取的镜像
docker rmi ghcr.io/nezhahq/nezha

🛡️ 安全与防火墙设置 (重要)

在 Ubuntu 22.04 上，如果启用了 ufw 防火墙，必须放行相关端口。

面板端服务器：

  • 8008 (或你设置的面板访问端口)：用于 Web 访问。

  • 5555 (或你设置的通信端口)：用于接收 Agent 数据。

sudo ufw allow 8008/tcp
sudo ufw allow 5555/tcp

被控端服务器：

通常不需要特殊设置入站规则，因为 Agent 是主动连接面板端的。

⚠️ 密钥安全警告

Agent 的安装命令中包含了 NZ_CLIENT_SECRET。

  1. 不要公开分享包含此密钥的完整安装命令。

  2. 如果怀疑密钥泄露，请立即在面板后台删除该服务器并重新添加，生成新的密钥，然后去被控端修改配置或重新安装。
