6.4 systemd-networkd-wait-online 超时（已处理）

  • 现象：重启后 systemd-networkd-wait-online.service 报红，提示 Timeout occurred while waiting for network connectivity.
  • 原因：当前实际网络由现有网络脚本与 Docker bridge 正常带起，但 networkctl 中相关接口显示为 unmanaged，wait-online 的等待条件与当前环境不匹配。
  • 处理：已执行
systemctl disable systemd-networkd-wait-online.service
systemctl stop systemd-networkd-wait-online.service
systemctl reset-failed systemd-networkd-wait-online.service
  • 当前影响：不影响 SSH、邮件、Nginx、Docker 正常工作。
  • 回滚：
systemctl enable systemd-networkd-wait-online.service
systemctl start systemd-networkd-wait-online.service

八、多域名邮箱扩展指南

当前 Postfix 已配置 virtual_mailbox_domains = qianyan.me, mail.qianyan.me，支持多域名。 Dovecot 的 mail_location = maildir:/var/mail/vmail/%d/%n 天然按域名隔离。

8.1 新增域名完整步骤（以 example.com 为例）

#### ① DNS 记录（在域名 DNS 控制台添加）

类型主机记录值
Amail 或 @74.48.40.123
MX@mail.example.com（优先级 10）
TXT@v=spf1 mx a ip4:74.48.40.123 ~all
TXTmail._domainkeyDKIM 公钥（由第③步生成）
TXT_dmarcv=DMARC1; p=quarantine; pct=100

#### ② 域名加入 Postfix

postconf -e virtual_mailbox_domains='qianyan.me, mail.qianyan.me, example.com'

#### ③ 生成 DKIM 密钥

mkdir -p /etc/opendkim/keys/example.com
cd /etc/opendkim/keys/example.com
opendkim-genkey -D /etc/opendkim/keys/example.com/ -d example.com -s mail
chown -R opendkim:opendkim /etc/opendkim/keys/example.com

#### ④ 注册 DKIM 到 KeyTable 和 SigningTable

编辑 /etc/opendkim/KeyTable：

mail._domainkey.example.com example.com:mail:/etc/opendkim/keys/example.com/mail.private

编辑 /etc/opendkim/SigningTable：

*@example.com mail._domainkey.example.com

#### ⑤ TLS 证书

方案 A（推荐）：将新域名作为 SAN 加入现有证书（使用现有续期钩子）：

certbot certonly --standalone --pre-hook 'systemctl stop nginx' \
  -d mail.qianyan.me -d example.com \
  --post-hook '/etc/letsencrypt/renewal-hooks/post/mail-reload.sh'

方案 B：为每个域名签发独立证书，在 Postfix/Dovecot 中配置 smtpd_tls_CAfile 合并。

#### ⑥ 重启服务

systemctl restart postfix dovecot opendkim
