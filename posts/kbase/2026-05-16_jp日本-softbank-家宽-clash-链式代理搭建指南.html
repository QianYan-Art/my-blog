核心逻辑在 /root/update_x-ui.core.sh：

  • 更新前自动备份 3x-ui 配置和服务文件到 /root/x-ui-update-backups/。
  • 使用 /root/cert/nat-jp-3h8e.aproxy.top/ 下的自签证书。
  • 证书不存在、过期或 30 天内到期时自动重建。
  • 更新时调用官方 update.sh，不调用 install.sh 冒充更新。
  • 不申请 Let's Encrypt 证书，不使用公网 80 端口。
  • 更新后等待 x-ui 和本地 HTTPS 恢复，避免把短暂 SQLite 锁误判为失败。

备份自动清理

2026-04-24 已部署每日自动清理任务，用于控制 cliproxyapi 和 3x-ui 更新脚本留下的备份数量：

/usr/local/sbin/cleanup-root-backups.sh
/etc/systemd/system/cleanup-root-backups.service
/etc/systemd/system/cleanup-root-backups.timer

cleanup-root-backups.service 是 oneshot 服务，只负责执行一次清理；cleanup-root-backups.timer 负责每日 05:40 JST 调度，已启用为 timers.target 下的 systemd timer。timer 配置了 Persistent=true，如果 VPS 在计划时间离线，恢复后会补跑一次。清理策略：

  • /root/cliproxyapi_backups/：仅保留最新 3 份。
  • /root/x-ui-update-backups/：仅保留最新 3 份。
  • /root/x-ui-backup-*：兼容历史旧备份目录，最多保留最新 3 个。
  • 脚本只对白名单路径执行删除，运行日志写入 journalctl -u cleanup-root-backups.service。

手动检查：

systemctl start cleanup-root-backups.service
systemctl status cleanup-root-backups.service --no-pager
systemctl list-timers cleanup-root-backups.timer --no-pager
journalctl -u cleanup-root-backups.service -n 80 --no-pager

🛠️ 第一部分：服务端搭建 (VPS)

1. 原理：NAT VPS 与端口转发

你购买的是一台 NAT VPS。与普通 VPS 不同，它没有独立的公网 IP，而是和几十个邻居共享一个公网 IP。

  • 入站 (你连它)：必须通过商家提供的端口映射（转发）。例如：公网 48005 -> 内网 10010。
  • 出站 (它连网)：它通过动态 IP（SoftBank 家宽）访问互联网，这正是 GPT 认为你是真人的关键。

2. 操作步骤

  1. 系统选择：Debian 11 (轻量稳定)。
  1. 初装面板：使用 3x-ui 一键脚本安装轻量化代理管理面板。后续维护不要反复跑安装脚本，更新统一使用本文前面的 /root/update_x-ui.sh update。
    bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
  1. 节点配置 (Shadowsocks)：

- 协议：Shadowsocks (简单、抗干扰强)。

- 加密：aes-256-gcm (关键点：之前用 Chacha20 导致 Xray 启动失败，改用这个最稳)。

- 端口：内网监听 36734 (示例)。

- 密码：强密码 (如 @Suyuan...)。

  1. 端口映射 (对齐)：

- 在 AAITR 后台，将公网端口（如 48005）转发到内网端口（36734）。

- 铁律：脚本填的 Port = AAITR 公网 Port；AAITR 转发目标 = 面板监听 Port。

💻 第二部分：客户端配置 (Clash Verge)

1. 原理：链式代理 (Relay)

Clash 的 Relay 模式允许流量“跳”两次。

  • 第一跳 (前置入口)：使用机场节点翻墙，连接到 AAITR 的入口服务器（因为 AAITR屏蔽了国内直连）。
  • 第二跳 (落地目标)：流量到达 AAITR 后，解密并从日本家宽出口发出。

2. 脚本逻辑解析 (JavaScript)

我们需要通过脚本（Script）动态修改 Clash 的配置文件，实现复杂的分组逻辑。

#### 核心功能模块：

  1. 垃圾过滤：自动剔除机场订阅中包含 "流量"、"官网"、"Emby" 等无效节点，防止自动测速被干扰。
  1. 自动优选 (⚡)：创建一个 url-test 组，每 300秒 自动测试所有机场节点，选出延迟最低的作为前置跳板。
  1. 全能链式 (🔗)：将 [前置入口组] 和 [落地目标组] 串联起来。
  1. UI 排序控制：通过数组重排，将最重要的组（主组、链式组）置顶显示。

🛡️ 第三部分：安全防护策略

对于这种暴露公网端口的机器，安全至关重要。我们采用了最彻底的“物理断网法”。

1. SSH 防护 (22端口)

  • 平时状态：在 AAITR 后台删除或暂停对内网 22 端口的转发规则。

- _原理_：黑客扫描公网 IP 时，根本找不到 SSH 入口，物理隔绝暴力破解。

  • 维护状态：需要管理时，临时加回转发规则，用完即删。

2. 代理防护

  • 使用 Shadowsocks + 强密码。
  • 由于面板本身并不暴露在公网（只暴露了 SS 端口），所以非常安全。
  • Fail2Ban：已卸载。因为采用了上述“关端口”策略，不再需要软件层面的防护占用内存。

📝 最终版 Clash Verge 脚本 (存档)

这是为你调试好的最终代码，集成了自动优选、垃圾过滤、自定义排序和双重节点支持。

function main(config, profileName) {
  config.proxies = config.proxies || [];
  config["proxy-groups"] = config["proxy-groups"] || [];

  // =======================================================
  // 1. 定义落地节点 (你的家宽 + 旧节点)
  // =======================================================
  var aaitrNode = {
    name: "🏠 AAITR-日本家宽",
    type: "ss",
    server: "nat-jp-3h8e.aproxy.top", // 公网入口域名
    port: 48005,                        // 公网入口端口 (需与转发规则一致)
    cipher: "aes-256-gcm",              // 加密方式 (需与面板一致)
    password: "",         // 密码
    udp: true
  };

  var myOldNodes = [
    { name: "♻️ US-Socks-1", type: "socks5", server: "38.79.47.229", port: 443, username: "", password: "", "skip-cert-verify": true, udp: true },
    { name: "♻️ JP-Socks-1", type: "socks5", server: "43.251.186.102", port: 443, username: "", password: "", "skip-cert-verify": true, udp: true },
    { name: "♻️ US-Socks-2", type: "socks5", server: "168.158.125.70", port: 12324, username: "", password: "", "skip-cert-verify": true, udp: true }
  ];

  // 加入节点到代理池
  var allTargets = [aaitrNode, ...myOldNodes];
  allTargets.forEach(node => {
    if (!config.proxies.find(p => p.name === node.name)) {
      config.proxies.push(node);
    }
  });

  // =======================================================
  // 2. 策略组构建
  // =======================================================
  
  // 2.1 垃圾词过滤与机场节点提取
  var excludeKeywords = ["剩余流量", "到期", "过期", "官网", "产品", "频道", "群", "重置", "防丢", "Emby", "Traffic", "Expire"];
  var targetNames = allTargets.map(n => n.name);
  
  var airportNodeNames = config.proxies
    .filter(p => {
      if (targetNames.includes(p.name)) return false; // 排除落地节点
      for (var k of excludeKeywords) {
        if (p.name.includes(k)) return false; // 排除垃圾关键词
      }
      return true;
    })
    .map(p => p.name);

  // 2.2 组建：前置自动优选 (⚡)
  var autoEntranceName = "⚡ 链式-前置自动";
  var autoEntranceGroup = {
    name: autoEntranceName,
    type: "url-test",
    url: "[http://www.gstatic.com/generate_204](http://www.gstatic.com/generate_204)",
    interval: 300,
    tolerance: 50,
    proxies: airportNodeNames
  };

  // 2.3 组建：手动入口选择 (🚀) - 包含自动组
  var entranceGroupName = "🚀 链式-前置入口";
  var entranceGroup = {
    name: entranceGroupName,
    type: "select",
    proxies: [autoEntranceName, ...airportNodeNames]
  };
  
  // 2.4 组建：落地目标选择 (🎯)
  var targetGroupName = "🎯 链式-落地目标";
  var targetGroup = {
    name: targetGroupName,
    type: "select",
    proxies: targetNames
  };

  // 2.5 组建：全能链式代理 (🔗) - 核心管道
  var relayGroupName = "🔗 全能链式代理";
  var relayGroup = {
    name: relayGroupName,
    type: "relay",
    proxies: [entranceGroupName, targetGroupName]
  };

  // =======================================================
  // 3. 界面排序控制
  // =======================================================
  
  var originalGroups = config["proxy-groups"];
  
  // 自动寻找订阅的主组 (通常含 ISP 或订阅名)
  var mainGroupIndex = originalGroups.findIndex(g => g.name.includes("HK-BEUP") || g.name.includes("ISP"));
  if (mainGroupIndex === -1) mainGroupIndex = 0;
  var mainGroup = originalGroups[mainGroupIndex];
  
  var otherGroups = originalGroups.filter((g, index) => index !== mainGroupIndex);

  // 重组顺序：主组 -> 入口 -> 落地 -> 链式 -> 自动 -> 其他
  config["proxy-groups"] = [
    mainGroup,          
    entranceGroup,      
    targetGroup,        
    relayGroup,         
    autoEntranceGroup,  
    ...otherGroups      
  ];

  // =======================================================
  // 4. 将链式组注入到选择列表中
  // =======================================================
  var groups = config["proxy-groups"];
  for (var j = 0; j < groups.length; j++) {
    var g = groups[j];
    if (g.type === "select" || g.type === "selector") {
      // 排除我们自己的组，防止死循环
      if ([entranceGroupName, targetGroupName, relayGroupName, autoEntranceName].includes(g.name)) {
        continue;
      }
      g.proxies = g.proxies || [];
      // 将链式代理置顶
      if (!g.proxies.includes(relayGroupName)) {
        g.proxies.unshift(relayGroupName);
      }
    }
  }

  return config;
}
