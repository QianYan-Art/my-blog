Komari 面板 Nginx 反向代理配置指南

2026-04-2712 min
c-仅记录

更新时间：2026-04-23

本文档基于当前服务器实际运行状态整理，适用于以下现状：

  • 源站服务器：103.97.176.178
  • Komari 面板容器监听：127.0.0.1 反代目标为宿主机 25774
  • 旧域名：komari.starryocean.qzz.io
  • 新域名：qianyan.4071253.xyz
  • 新域名通过 EdgeOne 访问，旧域名继续保留给已接入的探针

本文已移除旧版中不再使用的 agent.starryocean.qzz.io 双域名写法，改为当前实际可用的双入口方案。

1. 当前推荐架构

当前最稳的方式不是强行把所有探针切到新域名，而是保留两个域名长期共存：

旧探针 / 旧安装脚本 -> komari.starryocean.qzz.io -> 服务器 Nginx -> Komari
你日常访问面板      -> qianyan.4071253.xyz -> EdgeOne -> 源站 103.97.176.178 -> Nginx -> Komari

这样做的好处：

  • 已接入探针完全不受影响。
  • 新域名可以作为面板主入口使用。
  • 旧域名仍然是稳定兜底入口。
  • 不需要立刻改证书、重装探针或批量修改 agent 配置。

2. 当前服务器实际配置

2.1 Komari 容器

当前容器监听：

ghcr.io/komari-monitor/komari:latest
0.0.0.0:25774 -> 25774/tcp

关键环境变量：

KOMARI_LISTEN=0.0.0.0:25774
KOMARI_DB_TYPE=sqlite
KOMARI_DB_FILE=/app/data/komari.db

2.2 当前 Nginx 配置

当前线上实际只配置了旧域名：

配置文件：

/etc/nginx/sites-enabled/komari.starryocean.qzz.io
/etc/nginx/conf.d/komari_ws_origin.conf

当前有效配置：

server {
    listen 80;
    server_name komari.starryocean.qzz.io;

    if ($host = "103.97.176.178") {
        return 444;
    }

    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name komari.starryocean.qzz.io;

    ssl_certificate /etc/letsencrypt/live/komari.starryocean.qzz.io/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/komari.starryocean.qzz.io/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    client_max_body_size 50M;

    location ~ ^/api/(clients|rpc2)$ {
        proxy_pass http://127.0.0.1:25774;
        proxy_set_header Host $komari_ws_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $komari_connection_upgrade;
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }

    location / {
        proxy_pass http://127.0.0.1:25774;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

说明：

  • 当前源站 Nginx 不认识 qianyan.4071253.xyz。
  • 当前源站证书也只覆盖 komari.starryocean.qzz.io。
  • 这不是问题，因为新域名当前由 EdgeOne 回源时使用旧域名 Host/SNI。
  • /api/clients 和 /api/rpc2 是实时状态相关接口，单独使用 komari_ws_origin.conf 做可信 Origin 到 Host 的归一化。
  • qianyan.me、api.qianyan.me 不属于 Komari 面板入口，不应写入 Komari 的 WebSocket 白名单。

当前 WebSocket Origin 映射：

map $http_origin $komari_ws_host {
    default $host;
    ~^https://([A-Za-z0-9.-]+\.4071253\.xyz)$ $1;
    ~^https://(4071253\.xyz)$ $1;
    ~^https://(komari\.starryocean\.qzz\.io)$ $1;
}

map $http_upgrade $komari_connection_upgrade {
    default upgrade;
    '' close;
}

3. 当前证书状态

当前 Certbot 证书：

komari.starryocean.qzz.io

服务器上另有 api.qianyan.me 证书和 Nginx 站点，但那条链路属于 LM Studio / frp API 服务，不属于 Komari 面板。

当前 没有 为 qianyan.4071253.xyz 签发源站证书。

这意味着：

  • komari.starryocean.qzz.io 可以直接访问源站。
  • qianyan.4071253.xyz 不能直接带着自己域名去连源站，否则会证书不匹配。
  • 所以 EdgeOne 回源时必须继续使用 komari.starryocean.qzz.io 作为回源 Host/SNI。

4. 新域名 qianyan.4071253.xyz 的正确用法

4.1 EdgeOne 回源配置

新域名在 EdgeOne 中应按以下方式配置：

  • 源站地址：103.97.176.178
  • 回源协议：HTTPS
  • 回源端口：443
  • 回源 HOST 头：自定义为 komari.starryocean.qzz.io
  • 回源 SNI：komari.starryocean.qzz.io

这样做的原因：

  • 让 EdgeOne 直接回源到服务器 IP，不再依赖旧域名 DNS 做二次中转。
  • 同时保留旧域名证书兼容性，不需要立即改服务器证书。

4.2 EdgeOne 规则建议

匹配：

HOST 等于 qianyan.4071253.xyz

建议动作：

  • 缓存：不缓存，或遵循源站
  • 查询字符串：全部保留
  • WebSocket：开启
  • 最大连接超时时长：1800 秒
  • 图片处理 / 文件优化：关闭
  • HTTP/2：开启
  • IPv6：可按稳定优先策略关闭；若保留也能工作，因为 EdgeOne 会 IPv4 回源

注意：

  • EdgeOne 的访问域名是 qianyan.4071253.xyz，回源 Host/SNI 仍使用 komari.starryocean.qzz.io。
  • 浏览器发起 WebSocket 时，Origin 会是 https://qianyan.4071253.xyz，源站需要在 komari_ws_origin.conf 中显式信任该域名。
  • 如果以后新增面板入口域名，只加到 Komari 的 WebSocket Origin 映射，不要把无关的 qianyan.me 或 api.qianyan.me 放进去。

5. 旧域名 komari.starryocean.qzz.io 的使用原则

旧域名当前应保持以下策略：

  • 不删除
  • 不跳转到新域名
  • 不移除证书
  • 不停用 Nginx 对旧域名的监听

原因：

  • 已接入的探针仍通过旧域名上报
  • 本机 Komari Agent 也仍使用旧域名
  • Komari 当前生成安装脚本时，默认仍会写旧域名

6. 今天检查到的 Komari 关键状态

6.1 script_domain 仍是旧域名

从 Komari 数据库中可见：

script_domain = https://komari.starryocean.qzz.io

这意味着：

  • 你现在即使用新域名打开面板
  • 在“添加节点”里生成的安装脚本/下载链接，默认仍很可能使用旧域名

这是当前“保守稳定方案”的正常表现，不是错误。

6.2 本机 Komari Agent 也仍用旧域名

当前 systemd 服务：

/etc/systemd/system/komari-agent.service

当前启动参数：

ExecStart=/opt/komari/agent -e https://komari.starryocean.qzz.io ...

说明本机探针也仍按旧域名连接面板。

6.3 双域名访问都正常

本次检查结果：

  • https://komari.starryocean.qzz.io/：200 OK
  • 源站 Nginx 配置测试：通过

结论：

当前双域名方案工作正常，不需要为了“结构更漂亮”而立刻改服务器。

7. 是否要立即迁移到新域名

当前不建议强制全量迁移。

推荐策略：

  • 旧节点继续跑旧域名
  • 你自己用新域名访问面板
  • 等以后确实要迁移时，再分批替换 agent 端点

这是一种“可长期共存”的方案，不是临时过渡。

8. 如果以后要迁移到新域名

如果未来你希望新域名成为真正主域名，需要做的是：

8.1 修改 Komari 面板脚本域名

把 Komari 的 script_domain 改成：

https://qianyan.4071253.xyz

改完后，新节点通过面板“添加节点”获取到的安装脚本，就会默认使用新域名。

8.2 旧节点迁移方式

对已接入节点，不一定要重装。通常只要把 agent 的 -e 地址从旧域名改成新域名，再重启 agent 即可。

示例：

-e https://qianyan.4071253.xyz

8.3 真正“做干净”的最终方案

如果以后你想彻底摆脱旧域名依赖，需要再做两步：

  1. 给源站签一张包含 qianyan.4071253.xyz 的证书
  2. 让 Nginx server_name 同时支持两个域名

例如：

server {
    listen 443 ssl http2;
    server_name komari.starryocean.qzz.io qianyan.4071253.xyz;

    ssl_certificate /etc/letsencrypt/live/komari.starryocean.qzz.io/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/komari.starryocean.qzz.io/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:25774;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}

注意：

  • 真要这样做时，证书必须真的包含两个域名
  • 否则新域名直连源站仍会证书不匹配

9. 当前最稳建议

如果你不想折腾，当前最稳的做法就是：

保持不动

  • 保留旧域名 komari.starryocean.qzz.io
  • 保留旧域名证书
  • 保留旧域名 Nginx 配置
  • 保留 script_domain 为旧域名
  • 保留旧探针继续连接旧域名

继续使用

  • 你自己访问面板时使用：https://qianyan.4071253.xyz
  • 新域名通过 EdgeOne 回源到源站 IP
  • 回源 Host/SNI 使用旧域名

这套方案的核心优点是：

  • 稳
  • 不影响旧探针
  • 不需要批量重装或批量改 agent
  • 出现问题时旧域名仍可直接兜底

11. 当前文档结论

截至 2026-04-27：

  • 当前 Komari 的“双域名共存”方案是可用且推荐保留的。
  • 新域名已经可以正常作为面板入口。
  • 旧域名仍应继续保留给已有探针与安装脚本。
  • 服务器 Nginx 仅保留一段面板实时接口专用的 WebSocket Origin 兼容规则。
  • 现在不需要改证书或批量迁移探针。
  • 真正需要迁移时，再改 script_domain 与 agent -e 地址即可。
