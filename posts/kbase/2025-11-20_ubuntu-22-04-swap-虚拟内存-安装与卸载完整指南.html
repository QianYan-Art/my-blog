Ubuntu 22.04 Swap (虚拟内存) 安装与卸载完整指南

2025-11-203 min

本教程将指导您如何在 Ubuntu 22.04 系统上检查、创建、优化以及安全移除 Swap 文件。

1. 检查当前的 Swap 状态

在开始之前，首先检查系统是否已经配置了 Swap。

打开终端执行：

sudo swapon --show

或者：

free -h
  • 如果输出为空（swapon）或 Swap 一栏显示 0B，说明当前没有 Swap。
  • 如果已有 Swap，请根据需要决定是保留还是扩大（需先卸载旧的）。

2. 创建并启用 Swap (安装教程)

假设我们需要创建一个 4GB 的 Swap 文件。

第一步：创建 Swap 文件

我们将使用 fallocate 命令快速创建一个文件（比 dd 更快）。

sudo fallocate -l 4G /swapfile

> 注意：如果您需要不同大小，请将 4G 替换为 1G, 2G, 8G 等。

第二步：设置权限

为了安全起见，Swap 文件应该只能由 root 用户读取。

sudo chmod 600 /swapfile

第三步：将文件格式化为 Swap 区域

sudo mkswap /swapfile

第四步：激活 Swap

sudo swapon /swapfile

此时，您可以再次运行 sudo free -h，应该能看到 Swap 总量增加了。

第五步：设置开机自启 (持久化)

如果您现在重启电脑，刚才的设置会失效。我们需要将其写入 /etc/fstab 文件。

为了防止出错，先备份：

sudo cp /etc/fstab /etc/fstab.bak

将配置追加到文件末尾：

echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

3. 优化 Swap 性能 (可选推荐)

调整 Swappiness

swappiness 参数决定了系统使用 Swap 的频率。值范围 0-100。

  • 60 (默认)：适合桌面环境。
  • 10 (推荐服务器)：尽可能使用物理内存，仅在紧急时使用 Swap。

查看当前值：

cat /proc/sys/vm/swappiness

临时修改 (重启失效)：

sudo sysctl vm.swappiness=10

永久修改：

编辑配置文件：

sudo nano /etc/sysctl.conf

在文件末尾添加或修改以下行：

vm.swappiness=10

保存 (Ctrl+O, Enter) 并退出 (Ctrl+X)。

4. 删除 Swap (卸载教程)

如果您不再需要 Swap，或者想要调整大小（先删后加），请按以下步骤操作。

第一步：停用 Swap

这将把 Swap 中的数据移回内存，可能需要几秒钟。

sudo swapoff -v /swapfile

第二步：删除开机自启配置

我们需要从 /etc/fstab 中移除刚才添加的那一行，否则重启会报错。

打开文件：

sudo nano /etc/fstab

找到类似 /swapfile none swap sw 0 0 的行，将其删除或在行首加 # 注释掉。

保存并退出。

第三步：删除文件

最后，删除物理文件释放磁盘空间。

sudo rm /swapfile

常见问题

  1. Swap 应该设多大？

- 内存 < 2GB：Swap 设为内存的 2 倍。

- 内存 2GB - 8GB：Swap 等于内存大小。

- 内存 > 8GB：Swap 设为 4GB - 8GB 通常足够。

  1. 执行 fallocate 报错？

- 如果您的文件系统是 XFS 或某些老旧环境，fallocate 可能会失败。

- 替代方案：使用 dd 命令（速度较慢但更稳健）：

        sudo dd if=/dev/zero of=/swapfile bs=1G count=4
