外部调用方

-> 腾讯云新服务器 Nginx :443
-> 127.0.0.1:61423
-> frps 0.68.1
-> 另一台 Windows 工作机 frpc

关键点：

3. 新服务器端配置结论

3.1 frps

配置文件：

/etc/frp/frps.toml

当前关键配置：

bindPort = 7000
proxyBindAddr = "127.0.0.1"

auth.method = "token"
auth.token = "<redacted>"

transport.tls.force = true

说明：

  • bindPort = 7000：给远端 frpc 建控制连接。
  • proxyBindAddr = "127.0.0.1"：把业务端口 61423 限制在本机回环，仅允许 Nginx 本机回源。
  • transport.tls.force = true：要求 frpc 走 TLS。

这一步很关键。它直接避免了 61423 被公网绕过 Nginx 直接打到 frps。

3.2 Nginx

配置文件：

/etc/nginx/sites-available/lmstudio-api-shiqianyan.conf

当前关键配置：

server {
    listen 80;
    listen [::]:80;
    server_name api.shiqianyan.cn;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location / {
        return 301 https://$host$request_uri;
    }
}

server {
    listen 443 ssl;
    listen [::]:443 ssl;
    http2 on;
    server_name api.shiqianyan.cn;

    ssl_certificate /etc/letsencrypt/live/api.shiqianyan.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.shiqianyan.cn/privkey.pem;

    client_max_body_size 200m;
    client_body_timeout 1800s;
    send_timeout 1800s;

    proxy_connect_timeout 60s;
    proxy_send_timeout 1800s;
    proxy_read_timeout 1800s;

    proxy_buffering off;
    proxy_request_buffering off;
    add_header X-Accel-Buffering no always;

    location / {
        proxy_pass http://127.0.0.1:61423;
        proxy_http_version 1.1;
        proxy_set_header Host $proxy_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

本次确认并优化的点：

  • proxy_connect_timeout 调整为 60s，与旧优化文档保持一致。
  • 保留 proxy_buffering off、proxy_request_buffering off、X-Accel-Buffering no，减少 SSE/长响应卡顿风险。
  • 保留 http2 on。
  • 去掉 API 站点里固定写死的 WebSocket Upgrade/Connection 头。普通 OpenAI 兼容接口、图片上传、SSE 都不依赖这两个头，长期硬塞反而容易制造歧义。

3.3 证书

  • api.shiqianyan.cn 当前证书由 Let’s Encrypt 提供。
  • 80 端口仅用于证书挑战与跳转到 HTTPS。

---

4. 与旧优化原则的对应关系

旧文档里还成立的规则，在新服务器上继续保持：

  1. api.shiqianyan.cn 只做 API，不套普通网站缓存策略。
  2. 图片处理关闭。
  3. 不对 SSE/API 链路做 Gzip/Brotli 优化。
  4. HTTP/2 保持开启。
  5. 大请求和慢响应需要长超时。
  6. 如果将来视频上传超过 200MB，再按需上调 client_max_body_size。
  7. api.shiqianyan.cn 与服务器 IP 应继续加入本地代理软件直连规则，避免 TUN/fake-ip 影响。

5. 验证结果

2026-05-07 实查结果：

frps.service                       active

http://127.0.0.1:61423/v1/models   401, ~0.03s

解释：

  • 未带鉴权的 401 是正常结果，说明链路已经到达 API 鉴权层。
  • 带鉴权 200 说明 Nginx -> frps -> frpc -> LM Studio 整条链路可用。

6. 维护要求

后续维护这条链路时，优先按下面顺序排查：

  1. systemctl is-active frps
  2. ss -ltnp | grep 7000
  3. ss -ltnp | grep 61423
  4. nginx -t
  5. curl -k https://api.shiqianyan.cn/v1/models
  6. 带鉴权访问 https://api.shiqianyan.cn/v1/models
  7. 到另一台 Windows 工作机检查 frpc 与 LM Studio :1235

如果出现故障，优先分层判断：

  • 7000 不通：frps 或防火墙问题
  • 61423 没监听：frpc 未连上
  • 61423 通但域名不通：Nginx/证书/DNS 问题
  • 域名 401 正常但带鉴权失败：上游模型、参数或 LM Studio 问题
