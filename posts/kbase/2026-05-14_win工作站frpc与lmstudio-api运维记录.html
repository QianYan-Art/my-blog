五、LM Studio API 健康检查

5.1 背景

LM Studio 当前作为本机 API 服务上游，实际监听 0.0.0.0:1235，再由 frpc 转发到公网入口。虽然现阶段 API 可正常使用，但 LM Studio 日志中出现过 ERR_HTTP_HEADERS_SENT、MaxListenersExceededWarning 等应用层报错，因此补充一层轻量健康检查，用于判断“服务现在是否还能对外正常工作”。

5.2 远程机器当前状态（2026-05-14 检查）

项目结果
LM Studio 安装路径C:\Users\Administrator\AppData\Local\Programs\LM Studio\LM Studio.exe
LM Studio 版本0.4.12.0
API 监听端口1235
FRPC 服务状态Running
显卡NVIDIA GeForce RTX 3050 Laptop GPU 4GB
本机 API 轻量探活未带 token 请求 /v1/models 返回 401，判定为“接口在线且鉴权正常”

5.3 脚本落点与运行形式

为避免和 frp_0.67.0_windows_amd64 目录混放，单独建立运维目录：

C:\lmstudio-api-tools\
├─ check_lmstudio_api.ps1
└─ logs\
   └─ health_check.log

脚本职责：

  1. 检查 LM Studio.exe 进程是否存在
  2. 检查 FrpcTunnel 服务是否运行
  3. 检查 1235 端口是否处于监听状态
  4. 调用 lms server status 检查本地 API 服务状态
  5. 避免直接请求未鉴权 v1/models，防止 LM Studio 产生额外错误日志
  6. 将每次检查结果追加写入 health_check.log

5.4 计划任务注册结果

已注册 Windows 计划任务：

项目值
任务名LmStudioApiHealthCheck
运行账户SYSTEM
执行频率每 5 分钟
执行命令powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\lmstudio-api-tools\check_lmstudio_api.ps1"
日志文件C:\lmstudio-api-tools\logs\health_check.log

首次验证结果：

2026-05-14 19:37:08 | lmstudio_processes=6 | lmstudio_memory_mb=915.41 | frpc=Running | port_1235_listening=True | port_pid=10016 | api_healthy=True | status=0 | server_running_port_1235

判定：

  • api_healthy=True：说明本机 API 服务可达
  • status=0：说明 lms server status 返回正常
  • 这份日志比 LM Studio 原始日志更适合做日常巡检
  • 2026-05-14 首版健康检查曾直接请求未鉴权 v1/models，会在 LM Studio main.log 中触发 ERR_HTTP_HEADERS_SENT；后已改为 lms server status 方案，停止制造额外日志噪声

5.5 常用查看命令

远程查看最近 20 条健康日志：

Get-Content C:\lmstudio-api-tools\logs\health_check.log -Tail 20

远程手动执行一次健康检查：

powershell.exe -NoProfile -ExecutionPolicy Bypass -File C:\lmstudio-api-tools\check_lmstudio_api.ps1

查看计划任务状态：

Get-ScheduledTask -TaskName LmStudioApiHealthCheck
Get-ScheduledTaskInfo -TaskName LmStudioApiHealthCheck

5.6 回滚/移除方法

如需停用健康检查：

Unregister-ScheduledTask -TaskName LmStudioApiHealthCheck -Confirm:$false
Remove-Item C:\lmstudio-api-tools -Recurse -Force

六、LM Studio 配置审计与调整（2026-05-14）

6.1 检查结论

LM Studio 当前可用，但更像“桌面应用默认配置”，不是“稳定 API 节点配置”。主要风险点：

项目原值处理
developer.appUpdateChannelbeta已改为 stable
developer.allowDevelopmentPluginstrue已改为 false
developer.autoUpdateExtensionPackstrue已改为 false
autoLoadBundledLLMtrue已改为 false

处理原因：

  1. beta 更新通道不适合承担长期 API 服务，升级后行为变化风险更高。
  2. 开发插件能力对当前节点无必要，只会增加不确定性。
  3. 扩展包自动更新对纯 API 节点价值不高，反而可能带来后台变更。
  4. 自动加载内置模型不适合服务节点，会引入多余资源占用和启动噪声。

6.2 未改但已确认的项

项目当前值说明
backendDownloadChannelstable合理，保留
enableLocalServicefalse当前不依赖 headless local service，保留
defaultContextLength4096暂无证据表明需要统一改动
unloadPreviousJITModelOnLoadtrue对低资源机器更稳，保留
jitModelTTL.enabledtrue暂保留
modelLoadingGuardrails.modeoff暂未改；是否打开需结合你常用模型尺寸单独评估

6.3 日志问题判断

LM Studio 原始日志中多次出现：

  • ERR_HTTP_HEADERS_SENT
  • MaxListenersExceededWarning

当前判断：

  1. 这不是 frpc 链路故障。
  2. 至少一部分 ERR_HTTP_HEADERS_SENT 是由于未鉴权探测请求触发。
  3. 调整健康检查后，main.log 已停止因巡检继续增长。
  4. LM Studio 本身未暴露出清晰的“日志级别/日志关闭”设置项，实际可控的方式主要是减少触发条件、保持稳定通道、避免 dev/beta 特性。

6.7 LM Studio 设置建议

适合当前这台机器作为远程 API 节点的建议如下：

设置项建议说明
Require Authentication保持开启当前已开启，必须保留
Serve on Local Network如无局域网直连需求，建议关闭frpc 可直接转发本机 127.0.0.1:1235，不必额外暴露局域网
Enable CORS无浏览器前端调用需求则关闭降低暴露面
Allow per-request MCPs关闭当前是 API 节点，不需要
Allow calling servers from mcp.json关闭当前不需要把本机 MCP 暴露给 API 调用方
JIT Model Loading看用途决定单模型常驻建议关闭并手动预加载；多模型切换或低内存机器可保留开启
Auto Unload Unused JIT Models若启用 JIT，建议开启避免内存长期占满
Only Keep Last JIT Loaded Model若经常切模型，建议开启这台机器资源有限，更稳

当前不建议继续乱动的项：

  • defaultContextLength=4096：先保留，是否调大取决于你常用模型和显存压力
  • unloadPreviousJITModelOnLoad=true：保留
  • enableLocalService=false：保留
  • backendDownloadChannel=stable：保留

当前无法直接通过配置文件精确控制的项：

  • LM Studio 原始日志级别
  • 某些 Developer 页里的 Server 开关

这类设置更适合在 LM Studio 图形界面里人工核对一次。

七、电源与无线网卡策略收敛（2026-05-14）

7.1 检查结论

无线网络当前由 Tenda WiFi 6 Wireless USB Adapter 提供，链路在线。交流电下无线适配器电源模式已是 最高性能，交流电下 USB 选择性暂停 已禁用，整体适合长期联网。

真正有风险的是系统睡眠路径，而不是无线网卡本身：

  • 原 合上盖子 为 睡眠
  • 原 睡眠按钮 为 睡眠
  • 原 无人参与系统睡眠超时 为 120 秒
  • 原 混合睡眠 为 启用

这些设置会导致模型服务在无人值守时被中断。

7.2 已调整项

已将交流电场景下的关键设置调整为更适合服务节点的状态：

项目调整后
在此时间后睡眠从不
无人参与系统睡眠超时0
混合睡眠关闭
合上盖子操作不采取任何操作
睡眠按钮操作不采取任何操作
关闭显示器300 秒
无线适配器节能模式最高性能
USB 选择性暂停已禁用
USB 3 Link Power ManagementOff

说明：

  1. 显示器可以自动熄屏，但主机不睡眠。
  2. 合盖不会再把服务打断。
  3. USB 无线网卡不会因交流电省电策略被主动压低稳定性。

7.3 调整后服务验证

调整后已验证：

  • FrpcTunnel = Running
  • LM Studio 健康检查通过
  • 1235 监听正常
  • 当前监听地址为 127.0.0.1:1235

这说明当前架构已更接近理想形态：

frpc (同机) -> 127.0.0.1:1235 -> LM Studio

比监听 0.0.0.0 更收敛，也更安全。
