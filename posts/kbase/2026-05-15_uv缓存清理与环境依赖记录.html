后续如何恢复环境

情况 1：项目根目录已有 pyproject.toml / uv.lock

优先使用项目自己的声明文件恢复：

uv sync

如果项目使用 requirements.txt：

uv venv
uv pip install -r requirements.txt

情况 2：项目只有环境，没有正式依赖清单

使用本次补的 requirements.freeze.txt：

uv venv
uv pip install -r requirements.freeze.txt

如果不用 uv，也可以用标准 venv + pip：

python -m venv .venv
.venv\Scripts\python.exe -m pip install -r requirements.freeze.txt

关于 uv cache

本次清理对象主要包括：

  • archive-v0
  • sdists-v9
  • simple-v15
  • simple-v16
  • simple-v18
  • builds-v0
  • interpreter-v4
  • wheels-v5
  • environments-v2

这些目录属于：

  • 包下载缓存
  • 源码包缓存
  • 索引缓存
  • 构建缓存

清理后影响：

  1. 已经存在的本地虚拟环境一般仍可继续使用。
  2. 以后重新创建环境或补装依赖时，需要重新下载包。
  3. 离线安装能力会下降。

当前建议

  1. uv cache 可以清理。
  2. Docker 的 WSL 数据盘暂时不要动，除非确认不再需要本地 Docker 数据。
  3. 以后新建项目，尽量补齐以下至少一种依赖声明：

    4. - uv.lock - pyproject.toml - requirements.txt

  • 未删除任何项目源码
