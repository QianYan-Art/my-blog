1. 背景与目标

  • 本地 Windows 电脑使用 LM Studio 部署 GGUF 模型（含视觉能力），本地运行正常。

  • 使用 frp（本地 frpc + 云服务器 frps）将 LM Studio 的 OpenAI 兼容 API 暴露到公网云服务器。

  • 使用 api.qianyan.me 作为展示用途域名，希望“正式、稳定、可访问”，并且 图片输入（视觉）也能稳定工作。

  • 问题现象：纯文本聊天正常，但在 Cherry Studio 传图片时报 network error（或类似简单错误）。

2. 故障定位思路与关键结论

2.1 采用“三段对比法”定位故障层

我们用同一套请求体（含图片 base64）逐层测试：

  1. 直连云服务器 IP:61423

  2. 通过 https://api.qianyan.me 域名访问（当时还在 EdgeOne/边缘安全加速代理下）

  3. 对比结果

2.2 结论

  • 直连 http://云服务器IP:61423/v1/chat/completions：图片请求成功

  • 通过域名 https://api.qianyan.me/v1/chat/completions：报 524（Receive timeout from origin）

因此确认：

LM Studio 与 frp 隧道本身能支持视觉请求；真正导致“传图失败”的主要原因是 EdgeOne/边缘代理层回源超时或不兼容大请求/慢响应（视觉推理耗时明显更长）。

3. 解决方案总体架构

为保留“正式域名 + HTTPS”，同时绕开边缘代理导致的 524：

  • api.qianyan.me 改为 DNS Only（仅解析 A 记录），不再走 EdgeOne 七层代理。

  • 云服务器部署 Nginx：

    • 提供 https://api.qianyan.me 的 TLS 终端（Let’s Encrypt 证书）

    • 反向代理到本机 127.0.0.1:61423（frps 暴露端口）

    • 配置大请求体与长推理超时（视觉请求关键）

  • Cherry Studio / PowerShell 端改用 https://api.qianyan.me 访问

最终链路：

外部客户端（Cherry/脚本）
→ api.qianyan.me:443（Nginx）
→ 127.0.0.1:61423（frps 业务暴露端口）
→ frp 隧道
→ Windows 本机 LM Studio API

4. 服务器端操作记录（Ubuntu）

4.1 DNS 与连通性检查

在云服务器上确认域名解析与 HTTP 可访问：

  • 查看权威 NS 与 A 记录：

    • dig NS qianyan.me +short

    • dig A api.qianyan.me +short

    • dig @1.1.1.1 A api.qianyan.me +short

    • dig @8.8.8.8 A api.qianyan.me +short

  • 验证 80 端口通达性：

    • curl -I http://api.qianyan.me

    • 返回 401 是正常的（LM Studio API 需要 Authorization）

4.2 安装 Nginx 与 Certbot

  • 安装：

    • apt update

    • apt install -y nginx certbot python3-certbot-nginx

4.3 创建 ACME webroot 目录，并验证挑战路径不被反代

  • 创建并写入测试文件：

    • mkdir -p /var/www/certbot/.well-known/acme-challenge

    • echo OK-ACME > /var/www/certbot/.well-known/acme-challenge/ping

  • 验证：

    • curl http://api.qianyan.me/.well-known/acme-challenge/ping

    • 期望输出 OK-ACME

4.4 使用 webroot 方式签发 Let’s Encrypt 证书（成功）

  • 签发命令：

    • certbot certonly --webroot -w /var/www/certbot -d api.qianyan.me

  • 证书位置：

    • /etc/letsencrypt/live/api.qianyan.me/fullchain.pem

    • /etc/letsencrypt/live/api.qianyan.me/privkey.pem

4.5 配置 Nginx（80 跳转 HTTPS + 443 反代到 frps）

创建并启用站点配置文件：

  • 文件：/etc/nginx/sites-available/lmstudio-api.conf

  • 建立软链：/etc/nginx/sites-enabled/lmstudio-api.conf

核心点：

  • 80：

    • 保留 /.well-known/acme-challenge/ 走 webroot

    • 其余跳转到 HTTPS

  • 443：

    • 绑定证书

    • proxy_pass http://127.0.0.1:61423;

    • 工作站/本地 frpc 未在线时，127.0.0.1:61423 不会监听；当前 Nginx 已加入离线兜底：

      • proxy_intercept_errors on

      • error_page 502 503 504 = @lmstudio_offline

      • 返回 503 JSON：{"error":"lmstudio_upstream_offline", ...}

      • 这只改变上游离线时的响应格式；工作站开机、frpc 和 LM Studio API 正常时，仍按原链路反代。

    • 必须设置：

      • client_max_body_size（支持 base64 图片大请求）

      • proxy_read_timeout / proxy_send_timeout（视觉推理慢，避免 Nginx 超时）

      • proxy_buffering off、proxy_request_buffering off（避免缓冲引发卡顿/异常）

验证并 reload：

  • nginx -t

  • systemctl reload nginx

HTTPS 连通性验证：

  • curl -I https://api.qianyan.me

  • 返回 HTTP/2 401 属于正常现象（未带 Authorization）。

4.6 证书续期检查（成功）

  • systemctl status certbot.timer

  • certbot renew --dry-run（模拟续期成功）

5. 客户端（Windows）验证与排障记录

5.1 PowerShell OpenAI 兼容请求验证

  • 先用纯文本请求验证链路可用（携带 Authorization: Bearer）

  • 再用图片（base64 data URL）请求验证视觉可用

最终确认：

  • 直连 IP:61423：图片 OK

  • 域名 HTTPS：图片 OK（在 Nginx + DNS Only 方案下）

5.2 证书名不匹配问题（RemoteCertificateNameMismatch）

曾出现一次 RemoteCertificateNameMismatch，随后恢复正常。推断原因多为：

  • Windows DNS 缓存/旧解析尚未更新

  • 域名刚从代理模式切换到 A 记录直连，缓存短期不一致

处理方式：

  • ipconfig /flushdns

  • Clear-DnsClientCache

  • 再次执行请求后正常

（注意：-SkipCertificateCheck 仅可用于临时排障，不建议用于展示环境。）

6. frp 端口含义与当前状态确认

6.1 当前 frps 服务信息（云服务器）

  • frps 进程启动命令：

    • /usr/local/bin/frps -c /etc/frp/frps.toml

  • 监听端口检查：

    • ss -lntp | grep -E 'frps|:7000|:61423'

解释：

  • 7000：frps 控制端口（frpc 连接云服务器用，不能随意对公网完全关闭，否则隧道断）

  • 61423：由 frpc 配置的 remotePort 导致 frps 在服务端额外监听的业务端口（用来对外提供 LM Studio API 入口）

6.2 “61423 能关，7000 不能关”的原因

  • 61423 是业务暴露端口：外部访问应该统一走 443 → Nginx → 127.0.0.1:61423
    因此 61423 公网入站可以关闭（更安全、更正式）。

  • 7000 是 frpc 连入 frps 的控制端口：如果关掉，frpc 无法建立隧道，业务端口自然失效。
    因此 7000 必须可达（但建议限制来源 IP + 使用 token 等安全手段）。

6.3 今日安全加固动作

  • LM Studio API token 已更换（避免泄露风险）

  • 关闭 61423 的公网访问（只保留域名 HTTPS 入口）

（可选进一步加固：让 frps 业务端口只监听 127.0.0.1，以及限制可用端口白名单。）

7. 以后复用的检查清单（很实用）

7.1 一键检查服务是否通（服务器侧）

  • Nginx 是否正常：

    • nginx -t && systemctl reload nginx

  • HTTPS 是否可握手：

    • curl -I https://api.qianyan.me

  • frps 是否在：

    • systemctl status frps --no-pager

  • frps 监听端口：

    • ss -lntp | grep frps

7.2 Cherry Studio “传图失败”时快速定位

  • 先测试 PowerShell 直连域名发图（排除前端问题）

  • 看 Nginx 日志：

    • tail -n 100 /var/log/nginx/error.log

    • tail -n 50 /var/log/nginx/access.log

常见错误对策：

  • 413：调大 client_max_body_size

  • 504：调大 proxy_read_timeout

  • upstream closed：检查 frp/LM Studio 状态、资源与超时
